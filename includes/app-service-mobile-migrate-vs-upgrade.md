Existem duas opções para mover a partir dos serviços móveis para o serviço de aplicações: migração ou a atualização. Qualquer uma das opções proporciona acesso a todos os recursos de alojamento do serviço de aplicações:

* *Migrar* um serviço apenas altera o ambiente subjacente que aloja o seu back-end móvel. Ele requer **sem alterações de código** para o cliente móvel ou o projeto de servidor móveis. Se utilizar a opção de migração automatizada, Isto preserva suas **service.azure mobile.net** URL. 
* *Atualizando* um serviço faz **fazer alterações ao código** para o seu servidor e o projeto de cliente, mas permite-lhe tirar partido dos novos recursos SDK móveis, como melhorias para autenticação e mais flexibilidade para sua projeto de servidor.
