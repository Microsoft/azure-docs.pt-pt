# <a name="azure-active-directory-identity-protection-faq"></a>O Azure Active Directory Identity Protection FAQ

Este artigo contém respostas para perguntas mais frequentes sobre o Azure Active Directory (Azure AD) Identity Protection. Para obter mais informações, consulte [do Azure Active Directory Identity Protection](../active-directory-identityprotection.md). 


## <a name="why-do-some-risk-events-have-closed-system-status"></a>Por que motivo é que alguns eventos de risco tem o estado de "Fechado (sistema)"?

**R:** Estes eventos de risco foram detetados pelo Identity Protection e mais tarde fechados porque os eventos foram já não são considerados arriscados. Estes eventos não são considerados nível de risco do utilizador. 

---

## <a name="do-i-need-to-be-a-global-admin-to-use-identity-protection-in-the-azure-portal"></a>É necessário ser um administrador global para utilizar a proteção de identidade no portal do Azure?
**R:** Não. Pode ser um leitor de segurança, um administrador de segurança ou um Administrador Global para utilizar a proteção de identidade.

---

## <a name="how-do-i-get-identity-protection"></a>Como posso obter o Identity Protection?

**R:** Ver [introdução ao Azure Active Directory Premium](../fundamentals/active-directory-get-started-premium.md) uma resposta para essa pergunta.

---

## <a name="how-can-i-sort-users-in-users-flagged-for-risk"></a>Como posso classificar os utilizadores em "Utilizadores sinalizados para risco"?

**R:** Transferir os utilizadores sinalizados no relatório de risco clicando **baixe** na parte superior do **utilizadores sinalizados para risco** página. Em seguida, pode ordenar os dados transferidos com base nos campos disponíveis, incluindo atualizado último (UTC).
