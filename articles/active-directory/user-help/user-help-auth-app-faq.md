# <a name="microsoft-authenticator-app-faq"></a>Perguntas frequentes do aplicativo Microsoft Authenticator

Este artigo responde a perguntas comuns sobre o aplicativo Microsoft Authenticator. Se você não vir uma resposta para sua pergunta, vá para o [Fórum do aplicativo Microsoft Authenticator](https://social.technet.microsoft.com/Forums/en-US/home?forum=MicrosoftAuthenticatorApp).

O aplicativo Microsoft Authenticator substituiu o aplicativo Azure Authenticator e é o aplicativo recomendado quando você usa a autenticação multifator do Azure. O aplicativo Microsoft Authenticator está disponível para [Android](https://app.adjust.com/e3rxkc_7lfdtm?fallback=https%3A%2F%2Fplay.google.com%2Fstore%2Fapps%2Fdetails%3Fid%3Dcom.azure.authenticator) e [Ios](https://app.adjust.com/e3rxkc_7lfdtm?fallback=https%3A%2F%2Fitunes.apple.com%2Fus%2Fapp%2Fmicrosoft-authenticator%2Fid983156458).

## <a name="frequently-asked-questions"></a>Perguntas mais frequentes

| Pergunta | Solução |
| -------- | -------- |
| Quais dados o autenticador armazena em meu nome e como posso excluí-lo? | O aplicativo Microsoft Authenticator coleta três tipos de informações:<ul><li>Informações de conta que você fornece ao adicionar sua conta. Esses dados podem ser removidos removendo sua conta.</li><li>Dados de log de diagnóstico, que residem apenas no aplicativo até que você escolha **enviar logs** para a Microsoft por meio do menu **ajuda** do aplicativo. Esses arquivos de log contêm dados pessoais, como seus endereços de email (como alain@contoso.com,), endereços de servidor/IP e dados de dispositivo (como o nome do dispositivo e a versão do sistema operacional), com os dados pessoais limitados às informações necessárias para ajudar a solucionar problemas do aplicativo. Você pode exibir esses arquivos de log no aplicativo a qualquer momento para ver as informações que estão sendo coletadas. Se você enviar os arquivos de log, os engenheiros de aplicativo de autenticação poderão usá-lo para solucionar problemas relatados pelo cliente.</li><li>Dados de uso não identificáveis pessoalmente, como "iniciada adicionar fluxo de conta/conta adicionada com êxito" ou "notificação aprovada". Esses dados são parte integrante de nossas decisões de engenharia e nos ajudam a determinar quais recursos são importantes para você e onde os aprimoramentos precisam ser feitos na forma de atualizações para os aplicativos. Você, como um usuário de aplicativo, vê uma notificação dessa coleta de dados na primeira inicialização do aplicativo e é informado de que ele pode ser desativado na página de **configurações** do aplicativo. Você pode habilitar ou desabilitar essa configuração a qualquer momento.</li></ul> |
| Quais são os códigos no aplicativo? | Ao abrir o aplicativo Microsoft Authenticator, você verá suas contas adicionadas como blocos com um número de seis ou oito dígitos para cada uma.<br><br>Você usará esses códigos como verificação de que você está dizendo. Depois de entrar com seu nome de usuário e senha, você digitará o código de verificação que está no mesmo bloco que sua conta. Por exemplo, se você estiver Alain entrando em sua conta da Contoso, você usará o código de verificação 427303.<br><br>![Tela de contas no aplicativo](./media/user-help-auth-app-faq/auth-app-accounts.png) |
| Por que o número ao lado do código mantém a contagem inativa? | Você pode ver um temporizador de 30 segundos contando ao lado do código de verificação ativo. Esse temporizador é para que você nunca entre usando o mesmo código duas vezes. Ao contrário de uma senha, não queremos que você se lembre desse número. A ideia é que apenas alguém com acesso ao seu telefone saiba seu código. |
| Por que meu bloco de conta está cinza? | Algumas organizações exigem que o aplicativo Microsoft Authenticator funcione com o logon único e proteja os recursos organizacionais. Nessa situação, a conta não é usada para verificação em duas etapas e aparece como cinza ou inativa. Esse tipo de conta é frequentemente chamado de conta de "agente". |
| O que é o registro de dispositivo? | Sua organização pode querer que você registre o dispositivo para que ele possa entender quais dispositivos estão acessando recursos protegidos, como arquivos e aplicativos, e possivelmente ativar o acesso condicional para reduzir o risco de acesso impróprio a esses recursos. Você pode cancelar o registro do seu dispositivo usando **as configurações**, mas pode perder o acesso a emails no Outlook, arquivos no onedrive e perderá a capacidade de usar a entrada pelo telefone. |
| Preciso estar conectado à Internet ou à minha rede para obter e usar os códigos de verificação? | Os códigos não exigem que você esteja na Internet ou conectados a dados, portanto, você não precisa do serviço de telefone para entrar. Além disso, como o aplicativo para de ser executado assim que você fechá-lo, ele não drenará a bateria. |
| Só obtenho notificações quando o aplicativo está aberto. Se o aplicativo estiver fechado, não recebo notificações. | Se você estiver recebendo notificações, mas não um alerta, mesmo com o toque ativado, verifique as configurações do aplicativo. Verifique se o aplicativo está ativado para usar som ou para vibrar para notificações. Se você não receber notificações, verifique o seguinte:<ul><li>Seu telefone está no modo não incomodar ou silencioso? Esses modos podem impedir que aplicativos enviem notificações.</li><li>Você pode obter notificações de outros aplicativos? Caso contrário, pode ser um problema com as conexões de rede em seu telefone ou o canal de notificações do Android ou da Apple. Você pode tentar resolver suas conexões de rede por meio de suas configurações de telefone, mas talvez seja necessário conversar com seu provedor de serviços para ajudar com o canal de notificações do Android ou da Apple.</li><li>Você pode obter notificações para algumas contas no aplicativo, mas não outras? Em caso afirmativo, remova a conta problemática do seu aplicativo, adicione-a novamente, permitindo notificações, e veja se isso corrige o problema.</li></ul>Se você tentou todas essas etapas e ainda está tendo problemas, recomendamos enviar seus arquivos de log para diagnóstico. Abra o aplicativo, vá para **ajuda**e, em seguida, selecione **enviar logs**. Depois disso, vá para o [Fórum do aplicativo Microsoft Authenticator](https://social.technet.microsoft.com/Forums/en-US/home?forum=MicrosoftAuthenticatorApp) e conte-nos o problema que você está vendo e quais etapas você tentou até agora. |
| Estou usando os códigos de verificação no aplicativo, mas como faço para alternar para as notificações por push? | Você pode configurar isso para sua conta corporativa ou de estudante (se ativada pelo administrador) ou para sua conta Microsoft pessoal, mas as notificações não funcionarão para contas de terceiros, como Google ou Facebook.<br><br>Para mudar sua conta pessoal para notificações, você precisará registrar novamente seu dispositivo com a conta. Vá para **adicionar conta**, selecione **conta pessoal da Microsoft**e, em seguida, entre usando seu nome de usuário e senha.<br><br>Sua organização decide se deseja permitir notificações com um clique para sua conta corporativa ou de estudante, para que sua organização possa desativar esse recurso.|
|As notificações funcionam para contas que não são da Microsoft | Não, as notificações só funcionam com contas da Microsoft e contas de Azure Active Directory. Se seu trabalho ou escola usa contas do Azure AD, eles podem desativar esse recurso. |
| Recebi um novo dispositivo ou restaurei meu dispositivo a partir de um backup. Como fazer configurar minhas contas no aplicativo Microsoft Authenticator novamente? | Se você estiver executando um dispositivo iOS, ativou o **backup do icloud**e criou um backup de suas contas em seu dispositivo antigo; Você pode usar esse backup para recuperar as credenciais de sua conta no novo dispositivo. Para obter mais informações, consulte o artigo [backup e recuperação de credenciais de conta com o aplicativo Microsoft Authenticator](user-help-auth-app-backup-recovery.md) . |
| Perdi meu dispositivo ou movi para um novo dispositivo. Como fazer garantir que as notificações não continuem a ir para o dispositivo antigo? | Adicionar o aplicativo Microsoft Authenticator ao novo dispositivo não removerá automaticamente o aplicativo do dispositivo antigo. Até mesmo excluir o aplicativo do dispositivo antigo não é suficiente. Você deve excluir o aplicativo do dispositivo antigo e informar à Microsoft ou à sua organização para esquecer o dispositivo antigo e cancelar o registro de sua conta.<ul><li>**Para remover o aplicativo de um dispositivo usando um conta Microsoft pessoal.** Vá para a área de verificação em duas etapas da página de [segurança](https://account.microsoft.com/security) da conta e opte por desativar a verificação para o dispositivo antigo.</li><li>**Para remover o aplicativo de um dispositivo usando uma conta Microsoft corporativa ou de estudante.** Vá para a área de verificação em duas etapas da [](https://myapps.microsoft.com/) sua página myapps ou para o portal personalizado da sua organização e opte por desativar a verificação para o dispositivo antigo.</li></ul> |
| Como fazer remover uma conta do aplicativo? | <ul><li>**iOS.** Na tela principal, selecione o botão de menu e, em seguida, selecione **Editar contas**. Toque no ícone vermelho ao lado do nome da conta e toque em **remover conta**.</li><li>**Android.** Na tela principal, selecione o botão de menu e **edite contas**. Toque no **X** ao lado do nome da conta.</li></ul>Se você tiver um dispositivo registrado em sua organização, talvez seja necessário concluir uma etapa extra para remover sua conta. Nesses dispositivos, o aplicativo Microsoft Authenticator é registrado automaticamente como um administrador de dispositivo. Se desejar desinstalar completamente o aplicativo, primeiro você precisará cancelar o registro do aplicativo nas configurações do aplicativo. |
| Por que o aplicativo solicita tantas permissões? | Aqui está a lista completa de permissões que podem ser solicitadas e como elas são usadas pelo aplicativo. As permissões específicas que você vê dependerão do tipo de telefone que você tem.<ul><li>**Brinquedos.** Usado para digitalizar códigos QR quando você adiciona um trabalho, escola ou não conta Microsoft.</li><li>**Contatos e telefone.** O aplicativo requer essa permissão para que possa Pesquisar contas corporativas ou de estudante da Microsoft em seu telefone e adicioná-las ao aplicativo, ajudando a garantir que sua conta funcione corretamente. Essa permissão também ajuda a poupar tempo ao adicionar suas contas pessoais da Microsoft, preenchendo automaticamente algumas das informações para você, como seu nome e sobrenome.</li><li>**SMS.** Usado para garantir que seu número de telefone corresponda ao número no registro. Quando você entra com seu conta Microsoft pessoal pela primeira vez. Enviamos uma mensagem de texto para o telefone onde você baixou o aplicativo que inclui um código de verificação de 6-8 dígitos. Em vez de solicitar que você encontre esse código e insira-o no aplicativo, ele é encontrado na mensagem de texto para você.</li><li>**Desenhe em outros aplicativos.** A notificação que você obtém que verifica sua identidade também é exibida em qualquer outro aplicativo que possa estar em execução.</li><li>**Receber dados da Internet.** Essa permissão é necessária para enviar notificações.</li><li>**Impedir que o telefone entre em suspensão.** Se você registrar seu dispositivo na sua organização, sua organização poderá alterar essa política em seu telefone.</li><li>**Controle de vibração.** Você pode escolher se deseja uma vibração sempre que receber uma notificação para verificar sua identidade.</li><li>**Use hardware de impressão digital.** Algumas contas corporativas e de estudante exigem um PIN adicional sempre que você verifica sua identidade. Para facilitar o processo, permitimos que você use sua impressão digital em vez de inserir o PIN.</li><li> **Exibir conexões de rede.** Quando você adiciona um conta Microsoft, o aplicativo requer conexão de rede/Internet.</li><li>**Leia o conteúdo do seu armazenamento**. Essa permissão é usada apenas quando você relata um problema técnico por meio das configurações do aplicativo. Algumas informações do armazenamento são coletadas para diagnosticar o problema.</li><li>**Acesso total à rede.** Essa permissão é necessária para enviar notificações para verificar sua identidade.</li><li>**Executar na inicialização.** Se você reiniciar o telefone, essa permissão garantirá que você receba notificações para verificar sua identidade.</li></ul> |
| Por que o aplicativo Microsoft Authenticator permite que você aprove uma solicitação sem desbloquear o dispositivo? | Você não precisa desbloquear seu dispositivo para aprovar solicitações de verificação porque tudo o que você precisa provar é que você tem seu telefone com você. A verificação em duas etapas requer provar duas coisas: algo que você conhece e algo que você tem. A coisa que você sabe é sua senha. A coisa que você tem é seu telefone (configurada com o aplicativo Microsoft Authenticator e registrada como uma prova de MFA.) Portanto, ter o telefone e aprovar a solicitação atende aos critérios do segundo fator de autenticação. |
| Por que todas as minhas contas não aparecem quando abro o aplicativo Microsoft Authenticator no meu Apple Watch? | O aplicativo Microsoft Authenticator só dá suporte ao uso de contas pessoais ou de estudante ou corporativas da Microsoft com notificações por push no aplicativo Apple Watch Companion. Para suas outras contas, como o Google ou o Facebook, você precisará abrir o aplicativo autenticador em seu telefone para exibir seus códigos de verificação. |
| Por que não posso aprovar ou negar notificações no meu Apple Watch? | Primeiro, verifique se você atualizou para o aplicativo Microsoft Authenticator, versão 6.0.0 ou superior no seu iPhone. Depois disso, abra o aplicativo Microsoft Authenticator Companion no seu Apple Watch e procure por qualquer conta com um botão **Configurar** abaixo deles. Você deve concluir esse processo de configuração para aprovar as notificações para essas contas. |
| Estou recebendo um erro de comunicação entre o Apple Watch e o meu telefone. O que posso fazer para solucionar problemas? | Esse erro ocorre quando a tela Watch entra em suspensão antes de concluir a comunicação com seu telefone.<br><br><b>Se isso acontecer durante a instalação:</b><br>Tente executar a instalação novamente, certificando-se de manter seu relógio acordado até que o processo seja concluído. Ao mesmo tempo, abra o aplicativo em seu telefone e responda a quaisquer prompts que apareçam.<br><br>Se o telefone e o relógio ainda não estiverem se comunicando, você poderá tentar o seguinte:<ol><li>Forçar o encerramento do aplicativo Microsoft Authenticator telefone e abri-lo novamente no iPhone.</li><li>Forçar o encerramento do aplicativo complementar no seu Apple Watch.<ol><li> Abra o aplicativo Microsoft Authenticator Companion em seu relógio</li><li>Mantenha o botão lateral pressionado até que a tela de desligamento seja exibida.</li><li>Solte o botão lateral e mantenha pressionada a Digital Crown para forçar o encerramento do aplicativo ativo.</li></ol></li><li>Desative o Bluetooth e o Wi-Fi para o telefone e o relógio e, em seguida, ligue-os novamente.</li><li>Reinicie o iPhone e o relógio.</li></ol><b>Se isso acontecer quando você estiver tentando aprovar uma notificação:</b><br>Na próxima vez que você tentar aprovar uma notificação em seu Apple Watch, mantenha a tela ativa até que a solicitação seja concluída e ouça o som que indica que ela foi bem-sucedida. |
| Por que o aplicativo Microsoft Authenticator Companion para Apple Watch sincronizar ou aparecer em meu relógio? | Se o aplicativo não estiver aparecendo em seu relógio, tente o seguinte: <ol><li>Verifique se o relógio está executando o watchOS 4,0 ou superior.</li><li>Sincronize seu relógio novamente.</li></ol> |
| Meu aplicativo do Apple Watch Companion falhou. Posso enviar meus logs de falha para que você possa investigar? | Primeiro você precisa ter certeza de que optou por compartilhar sua análise conosco. Se você for um usuário do TestFlight, você já está inscrito. Caso contrário, você pode acessar **configurações > privacidade > Analytics** e selecionar as opções **compartilhar &** de vídeo do iPhone e **compartilhar com os desenvolvedores de aplicativos** .<br><br>Depois de se inscrever, você pode tentar reproduzir sua falha para que os logs de falhas sejam enviados automaticamente para nós para investigação. No entanto, se não for possível reproduzir sua falha, você poderá copiar manualmente os arquivos de log e enviá-los para nós.<ol><li>Abra o aplicativo Watch em seu telefone, vá para **configurações > geral**e clique em **copiar análise de inspeção**.</li><li>Localize a falha correspondente em **configurações > privacidade > análise > dados de análise**e, em seguida, copie manualmente o texto inteiro.</li><li>Abra o aplicativo Microsoft Authenticator em seu telefone e cole o texto copiado na caixa de texto **compartilhar com desenvolvedores de aplicativos** na página **enviar logs** .</li></ol> |
| O que é o recurso de bloqueio de aplicativo e como ele ajuda a me manter mais seguro? | Para manter suas senhas de uso único, informações de aplicativo e configurações de aplicativo mais seguras, você pode ativar o recurso de bloqueio de aplicativo no aplicativo Microsoft Authenticator. Ativar o bloqueio de aplicativo na tela **configurações** do aplicativo Microsoft Authenticator significa que você será solicitado a autenticar usando seu PIN ou biométrica toda vez que abrir o aplicativo Microsoft Authenticator. Esse recurso oferece proteção extra, a maneira como você aprova suas notificações no aplicativo Microsoft Authenticator não será alterada.<br><br>**Nota**<br>Se você retornar ao aplicativo Microsoft Authenticator em execução em um dispositivo Android em menos de 30 segundos, não será solicitado a autenticar novamente. |
| Por que estou recebendo notificações sobre a atividade da minha conta? | Para ajudar a manter você mais informado sobre o que está acontecendo com sua conta Microsoft pessoal, estamos enviando notificações de atividade para seu aplicativo Microsoft Authenticator. Essas notificações aparecem imediatamente após a alteração de algo, ajudando a manter você mais seguro. Anteriormente, enviamos essas notificações por email e SMS, e agora expandimos para incluir o aplicativo. Para obter mais informações sobre essas notificações de atividade, consulte [o que acontece se houver uma entrada incomum em sua conta](https://support.microsoft.com/help/13967/microsoft-account-unusual-sign-in). Para alterar onde você recebe suas notificações, entre na página [onde podemos entrar em contato com você com alertas de conta não críticos](https://account.live.com/SecurityNotifications/Update) de sua conta. |
