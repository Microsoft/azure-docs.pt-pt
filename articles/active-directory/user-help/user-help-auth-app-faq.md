# <a name="frequently-asked-questions-faqs-about-the-microsoft-authenticator-app"></a>Perguntas frequentes (PERGUNTAS) sobre a aplicação Microsoft Authenticator

Este artigo responde a perguntas comuns sobre a aplicação Microsoft Authenticator. Se não vir uma resposta à sua pergunta, vá ao fórum da [aplicação Microsoft Authenticator](https://social.technet.microsoft.com/Forums/en-US/home?forum=MicrosoftAuthenticatorApp).

A aplicação Microsoft Authenticator substituiu a aplicação Azure Authenticator, e é a aplicação recomendada quando utilizar a Autenticação Multi-Factor Azure. A aplicação Microsoft Authenticator está disponível para [Android](https://app.adjust.com/e3rxkc_7lfdtm?fallback=https%3A%2F%2Fplay.google.com%2Fstore%2Fapps%2Fdetails%3Fid%3Dcom.azure.authenticator) e [iOS.](https://app.adjust.com/e3rxkc_7lfdtm?fallback=https%3A%2F%2Fitunes.apple.com%2Fus%2Fapp%2Fmicrosoft-authenticator%2Fid983156458)

## <a name="frequently-asked-questions"></a>Perguntas mais frequentes

| Pergunta | Solução |
| -------- | -------- |
| Que dados o Autenticador armazena em meu nome e como posso apagá-lo? | A aplicação Microsoft Authenticator recolhe três tipos de informações:<ul><li>Informações de conta que fornece quando adiciona a sua conta. Estes dados podem ser removidos removendo a sua conta.</li><li>Dados de registo de diagnóstico, que residem apenas na aplicação até que escolha **enviar Registos** para a Microsoft através do menu **ajuda** da aplicação. Estes ficheiros de registo contêm dados pessoais, como os seus endereços de e-mail (como, alain@contoso.com), endereços servidor/IP e dados do dispositivo (como, por exemplo, nome do dispositivo e versão do sistema operativo), com os dados pessoais limitados a informações necessárias para ajudar a resolver problemas com as aplicações. Pode ver estes ficheiros de registo na aplicação a qualquer momento para ver a informação a ser recolhida. Se enviar os ficheiros de registo, os engenheiros da aplicação Authentication podem usá-lo para resolver problemas reportados pelo cliente.</li><li>Dados de utilização não pessoalmente identificáveis, tais "começaram a adicionar fluxo de conta/contas adicionadas com sucesso", ou "notificação aprovada". Estes dados são parte integrante das nossas decisões de engenharia e ajudam-nos a determinar quais as funcionalidades importantes para si, e onde é necessário fazer melhorias sob a forma de atualizações para as apps. Você, como utilizador de aplicações, vê uma notificação desta recolha de dados no primeiro lançamento da app, e é informado de que pode ser desligado na página de **Definições** da aplicação. Pode ativar ou desativar esta definição a qualquer momento.</li></ul> |
| Para que são os códigos na aplicação? | Ao abrir a aplicação Microsoft Authenticator, verá as suas contas adicionadas como azulejos com um número de seis ou oito dígitos para cada um.<br><br>Vais usar estes códigos como verificação de que és quem dizes ser. Depois de iniciar sessão com o seu nome de utilizador e senha, irá digitar o código de verificação que está no mesmo azulejo que a sua conta. Por exemplo, se estiver a assinar na sua conta Contoso, usaria o código de verificação, 427303.<br><br>![Ecrã de contas na app](./media/user-help-auth-app-faq/auth-app-accounts.png) |
| Porque é que o número ao lado do código continua a contar? | Pode ver um temporizador de 30 segundos a contar ao lado do seu código de verificação ativo. Este temporizador é para que nunca assines o mesmo código duas vezes. Ao contrário de uma senha, não queremos que se lembre deste número. A ideia é que só alguém com acesso ao seu telemóvel conheça o seu código. |
| Porque é que o meu azulejo de conta é cinzento? | Algumas organizações exigem que a aplicação Microsoft Authenticator trabalhe com um único sinal e proteja os recursos organizacionais. Nesta situação, a conta não é usada para verificação em duas etapas e aparece como cinzenta ou inativa. Este tipo de conta é frequentemente chamado de conta de "corretor". |
| O que é o registo do dispositivo? | A sua organização poderá querer que registe o dispositivo para que possa entender quais os dispositivos que estão a aceder a recursos seguros, como ficheiros e aplicações, e possivelmente ligar o Acesso Condicional para reduzir o risco de acesso inadequado a esses recursos. Pode desregistar o seu dispositivo utilizando **Definições,** mas poderá perder acesso a e-mails no Outlook, ficheiros no OneDrive e perderá a capacidade de utilizar o registo do telefone. |
| Preciso de estar ligado à Internet ou à minha rede para obter e usar os códigos de verificação? | Os códigos não requerem que esteja na Internet ou esteja ligado a dados, por isso não precisa de serviço telefónico para iniciar sessão. Além disso, como a aplicação deixa de funcionar assim que a fechar, não vai drenar a sua bateria. |
| Só recebo notificações quando a aplicação está aberta. Se a aplicação estiver fechada, não recebo notificações. | Se estiver a receber notificações, mas não um alerta, mesmo com o seu toque ligado, deverá verificar as definições da sua aplicação. Certifique-se de que a aplicação está ligada para usar som ou para vibrar para notificações. Se não receber notificações, deve verificar o seguinte:<ul><li>O seu telefone está em modo Não Perturbar ou Silencioso? Estes modos podem impedir que as aplicações enviem notificações.</li><li>Pode receber notificações de outras aplicações? Caso contrário, pode ser um problema com as ligações de rede no seu telemóvel ou com o canal de notificações do Android ou da Apple. Pode tentar resolver as ligações da sua rede através das definições do seu telefone, mas poderá ter de falar com o seu fornecedor de serviços para ajudar com o canal de notificações Android ou Apple.</li><li>Pode receber notificações de algumas contas na aplicação, mas não outras? Se sim, remova a conta problemática da sua app, adicione-a novamente permitindo notificações e veja se isso corrige o problema.</li></ul>Se tentou todos estes passos e ainda está com problemas, recomendamos o envio dos seus ficheiros de registo para diagnósticos. Abra a aplicação, vá à **Ajuda**e, em seguida, selecione **Enviar registos**. Depois disso, vá ao fórum de aplicações da [Microsoft Authenticator](https://social.technet.microsoft.com/Forums/en-US/home?forum=MicrosoftAuthenticatorApp) e informe-nos sobre o problema que está a ver e que passos tentou até agora. |
| Estou a usar os códigos de verificação na aplicação, mas como posso mudar para as notificações push? | Pode configurar isto para o seu trabalho ou conta escolar (se for ligado pelo seu administrador) ou para a sua conta pessoal da Microsoft, mas as notificações não funcionarão para contas de terceiros, como o Google ou o Facebook.<br><br>Para mudar a sua conta pessoal para notificações, terá de voltar a registar o seu dispositivo com a conta. Vá para **adicionar conta,** selecione **Conta Microsoft Pessoal**e, em seguida, inscreva-se usando o seu nome de utilizador e palavra-passe.<br><br>A sua organização decide se permite notificações de um clique para o seu trabalho ou conta escolar, para que a sua organização possa desativar esta funcionalidade.|
|As notificações funcionam para contas não Microsoft | Não, as notificações funcionam apenas com contas da Microsoft e contas azure Ative Directory. Se o seu trabalho ou escola usar contas Azure AD, podem desligar esta funcionalidade. |
| Comprei um dispositivo novo ou restaurei o meu dispositivo de uma cópia de segurança. Como configurar as minhas contas na aplicação Microsoft Authenticator de novo? | Se estiver a executar um iOS ou um dispositivo Android, e se tiver ligado o **Cloud Backup** no seu antigo dispositivo, pode utilizar o seu antigo backup para recuperar as credenciais da sua conta no seu novo dispositivo. Para mais informações, consulte as credenciais de conta de Backup e recupere as credenciais de conta com o artigo [da aplicação Microsoft Authenticator.](user-help-auth-app-backup-recovery.md) |
| Perdi o meu dispositivo ou mudei-me para um novo dispositivo. Como posso certificar-me de que as notificações não vão para o meu antigo dispositivo? | Adicionar a aplicação Microsoft Authenticator ao seu novo dispositivo não removerá automaticamente a aplicação do seu antigo dispositivo. Mesmo apagar a aplicação do seu antigo dispositivo não é suficiente. Ambos devem apagar a aplicação do seu antigo dispositivo e dizer à Microsoft ou à sua organização para esquecerem o dispositivo antigo e não o registarem na sua conta.<ul><li>**Para remover a aplicação de um dispositivo utilizando uma conta pessoal da Microsoft.** Vá para a área de verificação em duas etapas da sua [página de Segurança](https://account.microsoft.com/security) de Conta e opte por desativar a verificação do seu antigo dispositivo.</li><li>**Para remover a aplicação de um dispositivo que usa uma conta de trabalho ou escola da Microsoft.** Vá para a área de verificação em duas etapas da sua página [MyApps](https://myapps.microsoft.com/) ou para o portal personalizado da sua organização e opte por desativar a verificação do seu antigo dispositivo.</li></ul> |
| Como retiro uma conta da aplicação? | <ul><li>**IOS.** A partir do ecrã principal, selecione o botão menu e, em seguida, **selecione as contas Editar**. Toque no ícone vermelho ao lado do nome da conta e toque em **Remover a Conta**.</li><li>**Android.** A partir do ecrã principal, selecione o botão do menu **e,** em seguida, editar contas . Toque no **X** ao lado do nome da conta.</li></ul>Se tiver um dispositivo registado na sua organização, poderá ter de completar um passo extra para remover a sua conta. Nestes dispositivos, a aplicação Microsoft Authenticator é automaticamente registada como administradora de dispositivos. Se pretender desinstalar completamente a aplicação, tem de primeiro desregistar a aplicação nas definições da aplicação. |
| Por que a aplicação pede tantas permissões? | Aqui está a lista completa de permissões que podem ser pedidas, e como são usadas pela aplicação. As permissões específicas que vir dependerão do tipo de telefone que tem.<ul><li>**A câmara.** Usado para digitalizar códigos QR quando adiciona uma conta de trabalho, escola ou não Microsoft.</li><li>**Contactos e telefone.** A aplicação requer esta permissão para que possa pesquisar as contas existentes ou escolares da Microsoft no seu telemóvel e adicioná-las à aplicação, ajudando a garantir que a sua conta funciona corretamente. Esta permissão também ajuda a poupar tempo ao adicionar as suas contas pessoais da Microsoft, preenchendo automaticamente algumas das informações para si, como o seu primeiro e último nome.</li><li>**SMS.** Usado para ter certeza que o seu número de telefone corresponde ao número registado. Quando iniciar sessão com a sua conta pessoal da Microsoft pela primeira vez. Enviamos uma mensagem de texto para o telefone onde descarregou a aplicação que inclui um código de verificação de 6-8 dígitos. Em vez de lhe pedir para encontrar este código e insirí-lo na aplicação, encontra-se na mensagem de texto para si.</li><li>**Desenhe sobre outras aplicações.** A notificação que obtém que verifica a sua identidade também é exibida em qualquer outra aplicação que possa estar em execução.</li><li>**Receba dados da internet.** Esta permissão é necessária para o envio de notificações.</li><li>**Evite que o telefone durma.** Se registar o seu dispositivo com a sua organização, a sua organização pode alterar esta política no seu telemóvel.</li><li>**Controle a vibração.** Pode escolher se pretende uma vibração sempre que receber uma notificação para verificar a sua identidade.</li><li>**Use hardware de impressões digitais.** Alguns trabalhos e contas escolares requerem um PIN adicional sempre que verificar a sua identidade. Para facilitar o processo, permitimos que utilize a sua impressão digital em vez de introduzir o PIN.</li><li> **Ver ligações de rede.** Quando adiciona uma conta Microsoft, a aplicação requer ligação rede/internet.</li><li>**Leia o conteúdo do seu armazenamento.** Esta permissão só é utilizada quando reporta um problema técnico através das definições da aplicação. Algumas informações do seu armazenamento são recolhidas para diagnosticar o problema.</li><li>**Acesso total à rede.** Esta permissão é necessária para o envio de notificações para verificar a sua identidade.</li><li>**Corra na startup.** Se reiniciar o seu telemóvel, esta permissão garante que continua a receber notificações para verificar a sua identidade.</li></ul> |
| Porque é que a aplicação Microsoft Authenticator permite-lhe aprovar um pedido sem desbloquear o dispositivo? | Não precisa de desbloquear o seu dispositivo para aprovar pedidos de verificação porque tudo o que precisa de provar é que tem o telemóvel consigo. A verificação em duas etapas requer provar duas coisas : uma coisa que sabes, e uma coisa que tens. O que sabes é a tua senha. O que tem é o seu telefone (configurado com a aplicação Microsoft Authenticator e registado como prova de MFA.) Por isso, ter o telefone e aprovar o pedido satisfaz os critérios para o segundo fator de autenticação. |
| Porque é que todas as minhas contas não aparecem quando abro a aplicação do Autenticador Microsoft no meu Apple Watch? | A aplicação Microsoft Authenticator apenas suporta a utilização de contas pessoais ou escolares ou de trabalho da Microsoft com notificações push na aplicação de acompanhantes do Apple Watch. Para as suas outras contas, como o Google ou o Facebook, terá de abrir a aplicação autenticadora no seu telemóvel para ver os seus códigos de verificação. |
| Por que não posso aprovar ou negar notificações no meu Apple Watch? | Em primeiro lugar, certifique-se de ter atualizado para a aplicação Microsoft Authenticator, versão 6.0.0 ou superior no seu iPhone. Depois disso, abra a aplicação de acompanhante do Microsoft Authenticator no apple Watch e procure quaisquer contas com um botão **set Up** por baixo deles. Tem de completar esse processo de configuração para aprovar notificações dessas contas. |
| Estou a ter um erro de comunicação entre o Apple Watch e o meu telemóvel. O que posso fazer para resolver problemas? | Este erro ocorre quando o ecrã do Relógio vai dormir antes de terminar a comunicação com o telemóvel.<br><br><b>Se isto acontecer durante a configuração:</b><br>Tente voltar a executar a configuração, certificando-se de que mantém o Relógio acordado até que o processo esteja concluído. Ao mesmo tempo, abra a aplicação no seu telemóvel e responda a quaisquer indicações que apareçam.<br><br>Se o seu telefone e o Watch ainda não estiverem a comunicar, pode tentar o seguinte:<ol><li>A Força abandonou a aplicação de telefone microsoft Authenticator e abri-la novamente no seu iPhone.</li><li>A Força abandonou a aplicação de acompanhantes no apple watch.<ol><li> Abra a aplicação de acompanhante da Microsoft Authenticator no seu Relógio</li><li>Segure o botão lateral até aparecer o ecrã **de encerramento.**</li><li>Liberte o botão lateral e segure a Coroa Digital para forçar a saída da aplicação ativa.</li></ol></li><li>Desligue tanto bluetooth como Wi-Fi para o seu telefone e para o relógio, e depois volte a ligá-los.</li><li>Reinicie o iPhone e o relógio.</li></ol><b>Se isto acontecer quando estiver a tentar aprovar uma notificação:</b><br>Da próxima vez que tentar aprovar uma notificação no Apple Watch, mantenha o ecrã acordado até que o pedido esteja completo e ouça o som que indica que foi bem sucedido. |
| Porque é que a aplicação de acompanhante do Microsoft Authenticator não está a sincronizar ou a aparecer no meu relógio? | Se a aplicação não aparecer no relógio, tente o seguinte: <ol><li>Certifique-se de que o relógio está a executar o watchOS 4.0 ou superior.</li><li>Sincronize o relógio de novo.</li></ol> |
| A minha aplicação de acompanhante apple watch despenhou-se. Posso enviar-lhe os meus registos de acidente para que possa investigar? | Primeiro tens de te certificar que escolheste partilhar a tua análise connosco. Se é utilizador do TestFlight, já se inscreveu. Caso contrário, pode ir a **Definições > Privacy > Analytics** e selecionar tanto a análise do **iPhone & Watch share** como a Partilha com as opções de Desenvolvedores de **Aplicações.**<br><br>Depois de se inscrever, pode tentar reproduzir o seu acidente para que os seus registos de colisão sejam automaticamente enviados para nós para investigação. No entanto, se não conseguir reproduzir o seu crash, pode copiar manualmente os seus ficheiros de registo e enviá-los para nós.<ol><li>Abra a aplicação Watch no seu telefone, vá a **Definições > Geral,** e depois clique em **Copy Watch Analytics**.</li><li>Encontre o crash correspondente em **Definições > Privacidade > Analytics > Analytics Data,** e, em seguida, copie manualmente todo o texto.</li><li>Abra a aplicação Microsoft Authenticator no seu telefone e colhe que copiou texto para a Caixa de Texto **Com desenvolvedores** de aplicações na página **de registos enviar.**</li></ol> |
| Qual é a funcionalidade App Lock, e como é que ajuda a manter-me mais seguro? | Para manter as suas definições de senha única, informações de aplicações e configurações de aplicações mais seguras, pode ligar a funcionalidade App Lock na aplicação Microsoft Authenticator. Ligar o Bloqueio de Aplicações a partir do ecrã **definições** da aplicação Autenticador da Microsoft significa que será solicitado a autenticar usando o seu PIN ou biométrico sempre que abrir a aplicação Microsoft Authenticator. Esta funcionalidade oferece uma proteção extra, a forma como aprova as suas notificações na aplicação Microsoft Authenticator não se altera.<br><br>**Nota**<br>Como o registo do dispositivo pode acontecer em outros locais fora da aplicação Microsoft Authenticator, como na aplicação Portal da Empresa ou dentro das definições da conta Android, não há garantias de que o App Lock impeça um utilizador de aceder à Microsoft. Aplicação autenticadora. |
| Porque estou a receber notificações sobre a minha atividade de conta? | Para ajudar a mantê-lo mais informado sobre o que se passa com a sua conta pessoal da Microsoft, estamos a enviar notificações de atividade para a sua aplicação Microsoft Authenticator. Estas notificações aparecem imediatamente após a mudança de algo, ajudando a mantê-lo mais seguro. Anteriormente enviámos estas notificações por e-mail e SMS, e agora expandimos para incluir a app. Para mais informações sobre estas notificações de atividade, consulte [o que acontece se houver um inicio incomum na sua conta](https://support.microsoft.com/help/13967/microsoft-account-unusual-sign-in). Para alterar onde recebe as suas notificações, faça o inserição na página de [alertas](https://account.live.com/SecurityNotifications/Update) de conta não crítica da sua conta. |
| Ao iniciar sessão na minha conta de trabalho ou escola utilizando a aplicação de correio padrão que vem com o iOS, sou solicitado pela aplicação Microsoft Authenticator para as minhas informações de verificação de segurança. Depois de inserir essa informação e voltar para a aplicação de correio, tenho um erro. Que posso eu fazer? | Isto provavelmente acontece porque o seu sessão e a sua aplicação de correio estão a ocorrer em duas aplicações diferentes, fazendo com que o processo inicial de entrada de fundo deixe de funcionar e falhe. Para tentar corrigir isto, recomendamos que selecione o ícone **Safari** no lado inferior direito do ecrã enquanto faz sessão na sua aplicação de correio. Ao mudar-se para o Safari, todo o processo de inscrição acontece numa única aplicação, permitindo-lhe iniciar sessão na app com sucesso. |
| Os meus códigos de senha única (OTP) não estão a funcionar. O que devo fazer? | Certifique-se de que a data e a hora do seu dispositivo estão corretas e estão a ser automaticamente sincronizadas. Se a data e a hora estiverem erradas, ou dessincronizadas, o código não funcionará. |
| O sistema operativo Windows 10 Mobile foi deprecado em dezembro de 2019. Será que o Autenticador Microsoft nos sistemas operativos Windows Mobile também será depreciado? | A aplicação Microsoft Authenticator em todos os sistemas operativos Windows Mobile não será suportada após o dia 28 de fevereiro de 2020. Os utilizadores não serão elegíveis para receber quaisquer novas atualizações para a publicação da aplicação na data acima referida. Depois de 28 de fevereiro de 2020 os serviços da Microsoft que suportam atualmente autenticações utilizando o Autenticador Microsoft em todos os sistemas operativos Windows Mobile começarão a retirar o seu suporte. Para autenticar os serviços da Microsoft, encorajamos fortemente todos os nossos utilizadores a mudarem para um mecanismo de autenticação alternativo antes desta data. |
