# <a name="what-does-azure-multi-factor-authentication-mean-for-me"></a>O que o Multi-Factor Authentication do Azure significa para mim?
Então, recebeu um e-mail de alguém de TI ou o seu chefe diz que foi acrescentada uma verificação adicional de segurança à sua conta.  Em que é que isto se traduz?

Não se preocupe. A sua organização pretende apenas executar passos adicionais para garantir que é quem diz ser ao iniciar sessão. Esta verificação é feita através de uma combinação do seu nome de utilizador e palavra-passe e um telefone.  

Portanto, a primeira coisa a fazer é concluir o processo de registo.  Mas, antes de começarmos o processo, existem algumas coisas a decidir.

* **Telefone ou telemóvel do escritório** - escolha entre utilizar o telefone ou telemóvel do escritório.
* **Chamada de telemóvel, SMS ou aplicação móvel** - se estiver a utilizar um telemóvel, escolha entre receber uma chamada, SMS ou utilizar a aplicação móvel.
* **Aplicação móvel com um código de notificação ou verificação** - se estiver a utilizar a aplicação móvel, escolha entre receber uma notificação a que responde ou um código de verificação.

> [!NOTE]
> O suporte da sua empresa pode desativar uma ou mais destas opções.  Se assim for, essas opções não estarão disponíveis durante o processo de registo.  

Agora que decidiu o seu método de verificação, pode registar-se. Utilize a ligação [Experiência de Configuração](multi-factor-authentication-end-user-first-time.md) para começar com uma explicação passo a passo sobre o processo de inscrição.
