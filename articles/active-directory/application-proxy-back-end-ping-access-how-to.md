# <a name="how-to-configure-an-application-proxy-application-to-use-pingaccess"></a>Como configurar uma aplicação de Proxy de aplicações a utilizar PingAccess

A nossa colaboração com PingAccess agora permite-lhe expandir as vantagens do Proxy da aplicação para aplicações utilizando a autenticação baseada no cabeçalho. Se as suas aplicações não utilizarem cabeçalhos, consulte a nossa [Single Sign-On documentação](manage-apps/application-proxy-configure-single-sign-on-with-kcd.md) para obter detalhes sobre outras opções.

## <a name="overview-of-steps-and-recommended-documents"></a>Descrição geral dos passos e os documentos recomendados

Para configurar uma aplicação com PingAccess, existem quatro passos:

1.  Configurar Conetores da Proxy da aplicação

2.  Criar uma aplicação de Proxy de aplicações do Azure AD

3.  Transferir e configurar PingAccess

4.  Configurar aplicações na PingAccess

Para obter detalhes sobre cada um destes passos, consulte a nossa [-início de sessão único com a documentação de cabeçalhos](application-proxy-ping-access.md).
