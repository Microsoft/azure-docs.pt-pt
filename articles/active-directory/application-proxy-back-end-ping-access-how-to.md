# <a name="how-to-configure-an-application-proxy-application-to-use-pingaccess"></a>Como configurar uma aplicação de Proxy de aplicações para utilizar o PingAccess

Nossa colaboração com o PingAccess agora permite-lhe expandir os benefícios da Proxy da aplicação para aplicações através da autenticação baseada em cabeçalho. Se seus aplicativos não usar cabeçalhos, consulte nosso [documentação de início de sessão único](manage-apps/application-proxy-configure-single-sign-on-with-kcd.md) para obter detalhes sobre outras opções.

## <a name="overview-of-steps-and-recommended-documents"></a>Descrição geral de passos e os documentos recomendados

Para configurar uma aplicação com o PingAccess, existem quatro passos:

1.  Configurar conectores de Proxy de aplicação

2.  Criar uma aplicação de Proxy de aplicações do Azure AD

3.  Transferir e configurar o PingAccess

4.  Configurar aplicações no PingAccess

Para obter detalhes sobre cada uma dessas etapas, consulte nosso [início de sessão único com a documentação de cabeçalhos](manage-apps/application-proxy-configure-single-sign-on-with-ping-access.md).
