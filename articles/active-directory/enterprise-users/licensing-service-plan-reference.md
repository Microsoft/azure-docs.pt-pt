# <a name="product-names-and-service-plan-identifiers-for-licensing"></a>Nomes de produtos e identificadores de plano de serviço para licenciamento

Ao gerir licenças no [portal Azure](https://portal.azure.com/#blade/Microsoft_AAD_IAM/LicensesMenuBlade/Products) ou no [centro de administração microsoft 365,](https://admin.microsoft.com)vê nomes de produtos que se parecem com o Office *365 E3*. Quando utiliza powerShell v1.0 cmdlets, o mesmo produto é identificado usando um nome específico, mas menos amigável: *ENTERPRISEPACK*. Ao utilizar powerShell v2.0 cmdlets ou Microsoft Graph, o mesmo produto é identificado com um valor GUID: *6fd2c87f-b296-42f0-b197-1e91e994b900*. A tabela que se segue lista os produtos de serviço online da Microsoft mais utilizados e fornece os seus vários valores de ID. Estes quadros são para fins de referência e são precisos apenas a partir da data em que este artigo foi atualizado pela última vez. A Microsoft não planeia atualizá-los para serviços recém-adicionados periodicamente.

- **Nome do produto**: Usado em portais de gestão
- **ID de corda**: Usado por PowerShell v1.0 cmdlets ao realizar operações em licenças
- **GUID**: GUID : GUID utilizado pela Microsoft Graph API
- **Planos de serviço incluídos**: Uma lista de planos de serviço no produto que correspondem ao ID de cadeia e GUID
- **Planos de serviço incluídos (nomes amigáveis)**: Uma lista de planos de serviço (nomes amigáveis) no produto que correspondem ao ID de cadeia e ao GUID

## <a name="service-plans-that-cannot-be-assigned-at-the-same-time"></a>Planos de serviço que não podem ser atribuídos ao mesmo tempo

Alguns produtos contêm planos de serviço que não podem ser atribuídos ao mesmo utilizador ao mesmo tempo. Por exemplo, se tiver *o Office 365 E1* e *o Office 365 E3* no seu inquilino, e tentar atribuir ambas as licenças ao mesmo utilizador, a operação falha. Isto porque o produto E3 contém os seguintes planos de serviço que entram em conflito com os seus homólogos E1:

-   SharePoint Online (Plano 2) entra em conflito com o SharePoint Online (Plano 1).
-   Exchange Online (Plano 2) entra em conflito com o Exchange Online (Plano 1).

Ao utilizar o licenciamento baseado em grupo, experimenta [esta condição de erro.](licensing-groups-resolve-problems.md#conflicting-service-plans) Ao utilizar o PowerShell, vê o erro *de violência mutuamenteexclusiva.*

Esta secção lista os planos de serviço mais comuns que são mutuamente exclusivos, agrupados por tipo de serviço. Pode utilizar esta informação para planear a sua implementação de licença e evitar erros de atribuição. Estes quadros são para fins de referência e são precisos apenas a partir da data em que este artigo foi atualizado pela última vez. A Microsoft não planeia atualizá-los para serviços recém-adicionados periodicamente.
