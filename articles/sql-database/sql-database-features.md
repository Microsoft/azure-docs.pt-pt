# <a name="azure-sql-database-features"></a>Características da base de dados Azure SQL

O serviço de base de dados Azure SQL partilha uma base de código comum com a versão mais recente e estável do SQL Server. A maioria das funcionalidades padrão de linguagem SQL, processamento de consultas e gestão de bases de dados são idênticas no SQL Server e na Base de Dados Azure SQL. As funcionalidades que são comuns no Servidor SQL e todos os sabores da Base de Dados Azure SQL são:
  - Características [linguísticas - Controlo das palavras-chave da linguagem de fluxo](https://docs.microsoft.com/sql/t-sql/language-elements/control-of-flow), [Cursores,](https://docs.microsoft.com/sql/t-sql/language-elements/cursors-transact-sql)Tipos de [Dados,](https://docs.microsoft.com/sql/t-sql/data-types/data-types-transact-sql) [Declarações DML,](https://docs.microsoft.com/sql/t-sql/queries/queries) [Predicados, Números](https://docs.microsoft.com/sql/t-sql/queries/predicates)de [sequência,](https://docs.microsoft.com/sql/relational-databases/sequence-numbers/sequence-numbers) [procedimentos armazenados](https://docs.microsoft.com/sql/relational-databases/stored-procedures/stored-procedures-database-engine)e [Variáveis.](https://docs.microsoft.com/sql/t-sql/language-elements/variables-transact-sql)
  - Características da base de dados - [Afinação automática (plano de regulação)](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning), [Rastreio de alterações,](https://docs.microsoft.com/sql/relational-databases/track-changes/about-change-tracking-sql-server) [Colagem de bases](https://docs.microsoft.com/sql/relational-databases/collations/set-or-change-the-database-collation)de [dados, bases de dados contidas, utilizadores contidos,](https://docs.microsoft.com/sql/relational-databases/databases/contained-databases) [compressão](https://docs.microsoft.com/sql/relational-databases/data-compression/data-compression) [de](https://docs.microsoft.com/sql/relational-databases/security/contained-database-users-making-your-database-portable)dados, configurações de [configuração de bases](https://docs.microsoft.com/sql/t-sql/statements/alter-database-scoped-configuration-transact-sql)de dados, [operações de índice](https://docs.microsoft.com/sql/relational-databases/indexes/perform-index-operations-online)sonante, [partição](https://docs.microsoft.com/sql/relational-databases/partitions/partitioned-tables-and-indexes)e [tabelas temporais](https://docs.microsoft.com/sql/relational-databases/tables/temporal-tables) [(ver iniciar o guia).](sql-database-temporal-tables.md)
  - Funcionalidades de segurança - Funções de [aplicação,](https://docs.microsoft.com/sql/relational-databases/security/authentication-access/application-roles)máscara de [dados dinâmicos](https://docs.microsoft.com/sql/relational-databases/security/dynamic-data-masking) [(ver guia de início),](sql-database-dynamic-data-masking-get-started.md) [Segurança de Nível de Linha](https://docs.microsoft.com/sql/relational-databases/security/row-level-security)e Deteção de Ameaças - ver iniciar guias para bases de dados [únicas e piscinas elásticas](sql-database-threat-detection.md) e [instância gerida.](sql-database-managed-instance-threat-detection.md)
  - Capacidades multi-modelo - Processamento de [gráficos,](https://docs.microsoft.com/sql/relational-databases/graphs/sql-graph-overview) [dados JSON](https://docs.microsoft.com/sql/relational-databases/json/json-data-sql-server) [(ver guia de arranque),](sql-database-json-features.md) [OPENXML,](https://docs.microsoft.com/sql/t-sql/functions/openxml-transact-sql) [Espacial,](https://docs.microsoft.com/sql/relational-databases/spatial/spatial-data-sql-server) [OPENJSON](https://docs.microsoft.com/sql/t-sql/functions/openjson-transact-sql)e [xml índices](https://docs.microsoft.com/sql/t-sql/statements/create-xml-index-transact-sql).

A Base de Dados Azure SQL gere as suas bases de dados e garante a sua elevada disponibilidade. Algumas funcionalidades que podem afetar a alta disponibilidade ou não podem ser usadas no mundo paaS têm funcionalidades limitadas na Base de Dados Azure SQL. Além disso, algumas funcionalidades de base de dados dependem do tipo de Base de Dados Azure SQL que cria. Estas características são descritas nas tabelas abaixo. Com a Base de Dados Azure SQL, pode criar uma base de dados como parte de uma [instância gerida,](sql-database-managed-instance.md)como uma única base de dados, ou como parte de uma piscina elástica. Se precisar de mais detalhes sobre as diferenças, pode encontrá-las nas páginas separadas para bases de [dados individuais e piscinas elásticas](sql-database-transact-sql-information.md) ou [Instância Gerida](sql-database-managed-instance-transact-sql-information.md).

## <a name="migration-methods"></a>Métodos de migração

Pode utilizar diferentes métodos de migração para mover os seus dados entre o Servidor SQL, a Base de Dados Única e as bases de dados de Instâncias Geridas. Alguns métodos são **Online** e captam todas as alterações que são feitas na fonte enquanto você está executando a migração, enquanto em métodos **Offline** você precisa parar a sua carga de trabalho que está modificando dados sobre a fonte enquanto a migração está em andamento.

| **Origem** | **Base de dados única e piscina elástica** | **Piscinas de instâncias geridas e instâncias** |
| --- | --- | --- |
| Servidor SQL (on-prem, AzureVM, Amazon RDS) | **Online:** Serviço de Migração de [Dados (DMS)](https://docs.microsoft.com/sql/dma/dma-overview), [Replicação transacional](sql-database-managed-instance-transactional-replication.md) <br/> **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP | **Online:** Serviço de Migração de [Dados (DMS)](https://docs.microsoft.com/sql/dma/dma-overview), [Replicação transacional](sql-database-managed-instance-transactional-replication.md) <br/> **Offline:** Backup/restauro nativo, [ficheiro BACPAC (importação),](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database)BCP, [Replicação snapshot](sql-database-managed-instance-transactional-replication.md) |
| Base de dados individual | **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP | **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP |
| Instância Gerida | **Online:** [Replicação transacional](sql-database-managed-instance-transactional-replication.md) <br/> **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP, [Replicação snapshot](sql-database-managed-instance-transactional-replication.md) | **Online:** [Replicação transacional](sql-database-managed-instance-transactional-replication.md) <br/> **Offline:** Restauro de instância seleção transversal[(Azure PowerShell](https://docs.microsoft.com/powershell/module/az.sql/restore-azsqlinstancedatabase?#examples) ou [Azure CLI),](https://techcommunity.microsoft.com/t5/Azure-SQL-Database/Cross-instance-point-in-time-restore-in-Azure-SQL-Database/ba-p/386208) [backup/restauro nativo,](https://docs.microsoft.com/azure/sql-database/sql-database-managed-instance-get-started-restore) [ficheiro BACPAC (importação),](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database)BCP, [replicação snapshot](sql-database-managed-instance-transactional-replication.md) |
