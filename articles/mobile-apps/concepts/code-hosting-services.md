# <a name="cloud-hosted-source-code-management-services"></a>Serviços de gestão de código fonte hospedados na nuvem
Se tiver equipas de desenvolvimento com vários membros da equipa que trabalham na mesma base de código, precisa de encontrar um local adequado para hospedar o seu código. Armazenar os dados na nuvem e ter um repositório centralizado permite que todos carreguem, editem e gerem os ficheiros de código. Também podem interagir com outros desenvolvedores em projetos. O código pode ser facilmente acessível, independentemente de onde esteja, desde que tenha uma ligação à Internet.

Hospedar em nuvem é muito mais fácil do que opções no local. Requer menos configuração de hardware e permite que as organizações completem o processo de implementação de uma forma mais ágil.

## <a name="benefits-of-hosting-source-code-in-the-cloud"></a>Benefícios do código fonte de hospedagem na nuvem
- **Armazenamento centralizado em nuvem:** Veja e gere os seus dados a partir de qualquer lugar.
- **Melhor colaboração e código mais limpo:** Acompanhe o código dentro das equipas e gere projetos para garantir a entrega contínua de um excelente software.
- **Mais fácil de se envolver:** Contribua facilmente para os seus projetos.
- **Ciclo de libertação mais rápido:** Trabalhe mais rápido nas suas equipas e contribua facilmente para os seus projetos.
- **Reduzir o custo:** Não te preocupes em manter o teu próprio hardware, servidores, VPN, e assim por diante.

Utilize os seguintes serviços para hospedar os dados da sua aplicação na nuvem.

## <a name="github"></a>GitHub
[GitHub](https://github.com/) é um serviço de hospedagem de repositório de código aberto que acolhe projetos de código fonte em diferentes linguagens de programação. O GitHub acompanha as várias alterações feitas a cada iteração.

**Principais funcionalidades**
- Use o alojamento de código para manter todo o seu código num só local. Os repositórios privados, públicos e de código aberto estão todos equipados com ferramentas para o ajudar a hospedar, versão e código de libertação.
- Reveja o código e use as ferramentas de revisão incorporadas para fazer da revisão de código uma parte essencial do processo de qualquer equipa:
    - Proteja os balcões, proponha alterações e solicite revisões.
    - Diferenças de pontos, comentários em contexto, e obter feedback claro.
- Coordene cedo, mantenha-se alinhado, e faça mais com as ferramentas de gestão de projetos do GitHub:
    - Veja o panorama geral do projeto.
    - Utilize quadros de tarefas que estejam mesmo ao lado do seu código dentro do GitHub.
    - Arraste cartões para atribuir problemas ou puxe pedidos aos membros da equipa.
    - Estabeleça marcos para organizar e acompanhar o progresso.
    - Escreva notas para capturar ideias que possam ser úteis, mas que não pertençam a um determinado problema ou que puxem o pedido.
- Descubra e escolha facilmente as ferramentas certas para uma melhor comunicação e automatização do trabalho através da compra de aplicações no [GitHub Marketplace.](https://github.com/marketplace)
- Gerir e cultivar equipas utilizando: 
    - Funções de utilizador para ajudar a organizar equipas e permissões de acesso.
    - Ferramentas de linha de discussão para manter as conversas sobre o tema e a equipa focadas.
    - Diretrizes comunitárias para criar rapidamente novos membros da equipa com uma conta.
- Navegue e protagonize projetos populares para segui-los.
- Rede e aprender com os outros da indústria.

## <a name="azure-devops"></a>DevOps do Azure
[A Azure DevOps](https://azure.microsoft.com/services/devops/) suporta [o Azure Repos](https://azure.microsoft.com/services/devops/repos/) e [o Team Foundation Version Control (TFVC)](https://docs.microsoft.com/azure/devops/repos/tfvc/index?view=azure-devops) como opções de controlo de fontes. Possui repositórios privados gratuitos ilimitados com revisões de código colaborativas, gestão avançada de ficheiros, pesquisa de códigos e políticas de agências para garantir código de alta qualidade. O Azure Repos é ótimo para pequenos projetos e grandes organizações que precisam de apoio ao Azure Ative Directory nativo e políticas avançadas.
    
**Principais funcionalidades**
- Use um repositório de código de fonte git ilimitado para os seus repos públicos e privados:
    - Obter apoio para qualquer cliente Git.
    - Use ganchos web e integração API.
- Inicie a sua próxima construção a partir de um pedido de retirada de repo:
    - Colabore para construir um melhor código utilizando a discussão codificada e a integração contínua para cada mudança.
    - Configurar a integração contínua/entrega contínua (CI/CD) para ativar automaticamente as construções, testes e implementações com todos os pedidos de puxar concluídos. Pode utilizar a Azure Pipelines ou as suas ferramentas.
    - Proteja a qualidade do seu código com as políticas de ramificação.
- Mantenha o controlo centralizado da versão com o Team Foundation Version Control, que inclui revisão de código.
- Ligue-se ao seu código utilizando Xcode, Eclipse, IntelliJ, Android Studio, Visual Studio, Visual Studio Code e muito mais.
- Use uma poderosa pesquisa de código semântico.
- Obtenha benefícios com funcionalidades prontas para a empresa. A Azure DevOps tem integração nativa com o Azure Ative Directory, o que simplifica o processo de gestão do acesso aos seus repositórios de código.
- Garantir a qualidade do código com políticas de sucursais, tais como um número mínimo de revisões de código, requisitos para construções bem sucedidas e aplicação de estratégias de fusão de Git.
- Conecte o seu ambiente de desenvolvimento favorito para aceder a repos e gerir o trabalho.
