# <a name="different-stages-in-mobile-application-development"></a>Diferentes etapas no desenvolvimento de aplicações móveis
A construção de uma aplicação móvel envolve várias fases. Você pode facilmente aproveitar os serviços, ferramentas e tecnologias da Microsoft para construir aplicações móveis de alta classe.

## <a name="app-development"></a>Programação de aplicações
Você pode construir aplicações de clientes nativos de plataforma única com idiomas como Objective-C e Java. Pode construir aplicações multi-plataformas utilizando Xamarin, React Native e Unity. Também pode construir aplicativos híbridos usando a Cordova. Para construir aplicações de clientes, utilize IDEs e editores de código, tais como Visual Studio, Visual Studio for Mac ou Visual Studio Code em PCs e Macs.

## <a name="plan-and-design-with-back-end-services"></a>Plano e design com serviços back-end
 A Microsoft fornece uma variedade de serviços para a criação de uma parte traseira segura, escalável e fiável para a sua aplicação móvel. Pode criar APIs sem servidor e aciona sem ter de manter os seus próprios servidores. Utilize notificações push, sincronização de dados offline, autenticação e serviços de dados para formar uma parte traseira móvel abrangente e expansiva que suporta uma variedade de cenários de construção de aplicações. 
 
 A construção de um serviço back-end tem muitos requisitos. Tem de:
   - Limite o armazenamento de dados no dispositivo.
   - Sincronizar dados em vários dispositivos.
   - Certifique-se de que a aplicação funciona sem conectividade de rede.
   - Envie notificações aos utilizadores.
   - Reduza o consumo de baterias.
   - Construa uma parte traseira escalável, fiável e segura.
   - Automatizar a manutenção do servidor.

## <a name="devops-and-continuous-monitoring"></a>DevOps e monitorização contínua
Para criar aplicações móveis e focadas na nuvem, aproveite a integração contínua (CI), a entrega contínua (CD) e serviços de monitorização contínua para produzir construções nativas para todas as plataformas. Pode realizar testes de automação ui em milhares de dispositivos físicos, automatizar as suas versões através de canais de teste beta ou implementar diretamente nas lojas de aplicações. Também pode recolher informações de utilização, falha e exceção para as suas aplicações.

## <a name="additional-services"></a>Serviços adicionais
Muitos serviços Azure podem ser consumidos dependendo do tipo e natureza da aplicação que está sendo construída:
  - Pesquisa Cognitiva Azure para pesquisar todos os tipos de dados internos, com consulta rápida.
  - Azure Cognitive Services para usar inteligência artificial na sua aplicação.
  - Azure SignalR para ativar a comunicação em tempo real na sua aplicação.
