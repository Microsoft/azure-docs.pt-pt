# <a name="different-stages-in-mobile-application-development"></a>Diferentes fases no desenvolvimento de aplicações móveis
A construção de uma aplicação móvel envolve várias fases. Você pode facilmente aproveitar os serviços, ferramentas e tecnologias da Microsoft para construir aplicações móveis de alta classe.

## <a name="app-development"></a>Programação de aplicações
Você pode construir aplicações de clientes nativos de plataforma única com idiomas como Objective-C e Java. Você pode construir aplicativos cross-platform usando Xamarin, React Native e Unity. Também pode construir aplicações híbridas utilizando o Cordova. Para construir aplicações de clientes, faça uso de IDEs e editores de código, tais como Visual Studio, Visual Studio for Mac, ou Visual Studio Code em PCs e Macs.

## <a name="plan-and-design-with-back-end-services"></a>Plano e design com serviços de back-end
 A Microsoft fornece uma variedade de serviços para a criação de uma final segura, escalável e fiável para a sua aplicação móvel. Pode criar APIs e gatilhos sem ter de manter os seus próprios servidores. Utilize notificações push, sincronização de dados offline, autenticação e serviços de dados para formar uma extremidade móvel abrangente e expansiva que suporta uma variedade de cenários de construção de aplicações. 
 
 A construção de um serviço back-end tem muitos requisitos. Precisa:
   - Limite o armazenamento de dados no dispositivo.
   - Sincronizar dados em vários dispositivos.
   - Certifique-se de que a aplicação funciona sem conectividade de rede.
   - Envie notificações aos utilizadores.
   - Reduzir o consumo de bateria.
   - Construa uma extremidade escalável, fiável e segura.
   - Automatizar a manutenção do servidor.

## <a name="devops-and-continuous-monitoring"></a>DevOps e monitorização contínua
Para criar aplicações móveis e focadas na nuvem, aproveite a integração contínua (CI), entrega contínua (CD) e serviços de monitorização contínua para produzir construções nativas para todas as plataformas. Pode executar testes de automação de UI em milhares de dispositivos físicos, automatizar as suas libertações através de canais de testes beta ou implementar diretamente em lojas de aplicações. Também pode recolher informações de utilização, falha e de exceção para as suas aplicações.

## <a name="additional-services"></a>Serviços adicionais
Muitos serviços Azure podem ser consumidos dependendo do tipo e natureza da aplicação que está sendo construída:
  - Pesquisa Cognitiva Azure para pesquisar todos os tipos de dados internos, com consulta rápida.
  - Serviços Cognitivos Azure para usar inteligência artificial na sua aplicação.
  - O Azure SignalR permite a comunicação em tempo real na sua aplicação.
