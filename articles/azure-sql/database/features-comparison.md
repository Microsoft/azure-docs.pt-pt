# <a name="features-comparison-azure-sql-database-and-azure-sql-managed-instance"></a>Comparação de características: Base de Dados Azure SQL e Instância Gerida Azure SQL
[!INCLUDE[appliesto-sqldb-sqlmi](../includes/appliesto-sqldb-sqlmi.md)]

A Base de Dados Azure SQL e a SQL Manged Instance partilham uma base de código comum com a versão mais recente e estável do SQL Server. A maioria das funcionalidades padrão de linguagem SQL, processamento de consultas e gestão de bases de dados são idênticas. As funcionalidades que são comuns entre o Servidor SQL e a Base de Dados SQL ou a Instância Gerida SQL são:

- Características [linguísticas - Controlo das palavras-chave da linguagem de fluxo](https://docs.microsoft.com/sql/t-sql/language-elements/control-of-flow), [Cursores,](https://docs.microsoft.com/sql/t-sql/language-elements/cursors-transact-sql)Tipos de [Dados,](https://docs.microsoft.com/sql/t-sql/data-types/data-types-transact-sql) [Declarações DML,](https://docs.microsoft.com/sql/t-sql/queries/queries) [Predicados, Números](https://docs.microsoft.com/sql/t-sql/queries/predicates)de [sequência,](https://docs.microsoft.com/sql/relational-databases/sequence-numbers/sequence-numbers) [procedimentos armazenados](https://docs.microsoft.com/sql/relational-databases/stored-procedures/stored-procedures-database-engine)e [Variáveis.](https://docs.microsoft.com/sql/t-sql/language-elements/variables-transact-sql)
- Características da base de dados - [Afinação automática (plano de regulação)](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning), [Rastreio de alterações,](https://docs.microsoft.com/sql/relational-databases/track-changes/about-change-tracking-sql-server) [Colagem de bases](https://docs.microsoft.com/sql/relational-databases/collations/set-or-change-the-database-collation)de [dados, bases de dados contidas, utilizadores contidos,](https://docs.microsoft.com/sql/relational-databases/databases/contained-databases) [compressão](https://docs.microsoft.com/sql/relational-databases/data-compression/data-compression) [de](https://docs.microsoft.com/sql/relational-databases/security/contained-database-users-making-your-database-portable)dados, configurações de [configuração de bases](https://docs.microsoft.com/sql/t-sql/statements/alter-database-scoped-configuration-transact-sql)de dados, [operações de índice](https://docs.microsoft.com/sql/relational-databases/indexes/perform-index-operations-online)sonante, [partição](https://docs.microsoft.com/sql/relational-databases/partitions/partitioned-tables-and-indexes)e [tabelas temporais](https://docs.microsoft.com/sql/relational-databases/tables/temporal-tables) [(ver iniciar o guia).](../temporal-tables.md)
- Funcionalidades de segurança - [Funções](https://docs.microsoft.com/sql/relational-databases/security/authentication-access/application-roles)de aplicação , [Máscara de dados dinâmicos](https://docs.microsoft.com/sql/relational-databases/security/dynamic-data-masking) [(ver guia de início),](dynamic-data-masking-overview.md) [Segurança de Nível de Linha](https://docs.microsoft.com/sql/relational-databases/security/row-level-security)e Deteção de Ameaças - ver obter guias de arranque para base de dados [SQL](threat-detection-configure.md) e [Instância Gerida SQL](../managed-instance/threat-detection-configure.md).
- Capacidades multi-modelo - Processamento de [gráficos,](https://docs.microsoft.com/sql/relational-databases/graphs/sql-graph-overview) [dados JSON](https://docs.microsoft.com/sql/relational-databases/json/json-data-sql-server) [(ver guia de arranque),](json-features.md) [OPENXML,](https://docs.microsoft.com/sql/t-sql/functions/openxml-transact-sql) [Espacial,](https://docs.microsoft.com/sql/relational-databases/spatial/spatial-data-sql-server) [OPENJSON](https://docs.microsoft.com/sql/t-sql/functions/openjson-transact-sql)e [xml índices](https://docs.microsoft.com/sql/t-sql/statements/create-xml-index-transact-sql).

O Azure gere as suas bases de dados e garante a sua elevada disponibilidade. Algumas funcionalidades que podem afetar a alta disponibilidade ou não podem ser usadas no mundo paaS têm funcionalidades limitadas na Base de Dados SQL e na Instância Gerida SQL. Estas características são descritas nas tabelas abaixo. Se precisar de mais detalhes sobre as diferenças, pode encontrá-las nas páginas separadas para a Base de [Dados Azure SQL](../managed-instance/transact-sql-tsql-differences-sql-server.md) ou para a [Instância Gerida Azure SQL](../managed-instance/transact-sql-tsql-differences-sql-server.md).

## <a name="migration-methods"></a>Métodos de migração

Pode utilizar diferentes métodos de migração para mover os seus dados entre o SQL Server, o Azure SQL Database e o Azure SQL Managed Instance. Alguns métodos são **Online** e captam todas as alterações que são feitas na fonte enquanto você está executando a migração, enquanto em métodos **Offline** você precisa parar a sua carga de trabalho que está modificando dados sobre a fonte enquanto a migração está em andamento.

| **Origem** | **Base de Dados SQL do Azure** | **Instância Gerida do Azure SQL** |
| --- | --- | --- |
| Servidor SQL (on-prem, AzureVM, Amazon RDS) | **Online:** Serviço de Migração de [Dados (DMS)](https://docs.microsoft.com/sql/dma/dma-overview), [Replicação transacional](../managed-instance/replication-transactional-overview.md) <br/> **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP | **Online:** Serviço de Migração de [Dados (DMS)](https://docs.microsoft.com/sql/dma/dma-overview), [Replicação transacional](../managed-instance/replication-transactional-overview.md) <br/> **Offline:** Backup/restauro nativo, [ficheiro BACPAC (importação),](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database)BCP, [Replicação snapshot](../managed-instance/replication-transactional-overview.md) |
| Base de dados individual | **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP | **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP |
| Instância Gerida do SQL | **Online:** [Replicação transacional](../managed-instance/replication-transactional-overview.md) <br/> **Offline:** [ficheiro BACPAC (importação)](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database), BCP, [Replicação snapshot](../managed-instance/replication-transactional-overview.md) | **Online:** [Replicação transacional](../managed-instance/replication-transactional-overview.md) <br/> **Offline:** Restauro de instância seleção transversal[(Azure PowerShell](https://docs.microsoft.com/powershell/module/az.sql/restore-azsqlinstancedatabase?#examples) ou [Azure CLI),](https://techcommunity.microsoft.com/t5/Azure-SQL-Database/Cross-instance-point-in-time-restore-in-Azure-SQL-Database/ba-p/386208) [backup/restauro nativo,](https://docs.microsoft.com/azure/sql-database/sql-database-managed-instance-get-started-restore) [ficheiro BACPAC (importação),](https://docs.microsoft.com/sql/relational-databases/data-tier-applications/import-a-bacpac-file-to-create-a-new-user-database)BCP, [replicação snapshot](../managed-instance/replication-transactional-overview.md) |
