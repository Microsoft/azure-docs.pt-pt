# <a name="azure-vmware-solution-avs-upgrade-concepts"></a>Conceitos de atualização da Azure VMware Solution (AVS)

Um dos principais benefícios das nuvens privadas azure VMware Solution (AVS) é que a plataforma é mantida para si. A manutenção da plataforma inclui upgrades automatizados para um pacote de software validado vMware. As atualizações são feitas regularmente, garantindo que tem sempre as versões mais recentes validadas do software.

## <a name="avs-private-cloud-software-upgrades"></a>Atualizações de software de nuvem privada AVS

A plataforma privada de nuvem AVS inclui versões específicas do vSphere VSphere, ESXi, vSAN e NSX-T software. O pacote privado de software em nuvem é validado para uso em novas instalações privadas em nuvem, e para upgrades de nuvens privadas existentes.

O processo de atualização da gestão do ciclo de vida não requer tempo de inatividade para as suas nuvens privadas. O processo de upgrade garante que está a utilizar automaticamente a versão mais recente do software de nuvem privada AVS validado. As atualizações são aplicadas em cadência regular para que as nuvens privadas nunca sejam mais do que uma versão por trás do mais recente lançamento do pacote de software validado. Será notificado das atualizações planeadas para a sua nuvem privada. Pode adiar a atualização se a sua nuvem privada estiver dentro de uma versão do mais recente lançamento.

Patches e atualizações críticas são aplicados quando são validados. Será notificado antes das atualizações críticas necessárias. Esta política garante que a sua nuvem privada tem patches e atualizações críticas aplicadas imediatamente.

As versões de software VMware são fornecidas no artigo de conceito de [nuvens privadas e clusters](concepts-private-clouds-clusters.md) e no [FAQ](faq.md).
