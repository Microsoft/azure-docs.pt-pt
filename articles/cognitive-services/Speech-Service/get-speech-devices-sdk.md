# <a name="get-the-cognitive-services-speech-devices-sdk"></a>Obter os dispositivos de voz dos serviços cognitivos SDK

## <a name="requesting-access"></a>Pedir acesso

O SDK de dispositivos de voz está em pré-visualização restrita e tem de estar inscritos no programa. Atualmente, a Microsoft prefere grandes empresas como candidatos para o acesso para este produto.

Para obter acesso para o SDK de dispositivos de voz, siga estes passos:

1. Vá para o SDK de dispositivos do Microsoft Speech [formulário de inscrição](https://aka.ms/sdsdk-signup).
1. Leitura a [contrato de licença](speech-devices-sdk-license.md).
1. Se concordar com os termos do presente contrato de licença, selecione "Concordo".
1. Responda às perguntas no formulário.
1. Submeta o formulário. 
1. Se seu endereço de e-mail já não faz parte do Azure Active Directory, receberá um e-mail de convite, como o mostrado abaixo, após a aprovação. Se o seu endereço de e-mail já está no Azure Active Directory, receberá uma mensagem de e-mail da equipa do Microsoft Speech após a aprovação e pode avançar diretamente para [transferir o SDK de dispositivos de voz](#download-the-speech-devices-sdk).

## <a name="approval-e-mail"></a>E-mail de aprovação

```
From: Microsoft Speech Team from Microsoft (via Microsoft) <invites@microsoft.com> 
Subject: You're invited to the Microsoft organization 
```

![mensagem de e-mail](media/speech-devices-sdk/get-sdk-1.png)

## <a name="accept-access"></a>Aceitar o acesso
Execute os passos abaixo para aderir ao Azure Active Directory com o endereço de e-mail que indicou durante o registo. Este processo lhe concede acesso para o SDK de dispositivos de voz [site de download](https://shares.datatransfer.microsoft.com/).

1. Clique em **começar** na mensagem de e-mail que recebeu. Se sua organização já é cliente do Office 365, será solicitado a iniciar sessão e poderá avançar diretamente para o passo 8.

2. Clique em **seguinte** na janela do browser iniciado.

    ![janela de autenticação](media/speech-devices-sdk/get-sdk-2.png)

3. Crie uma conta Microsoft, se ainda não tiver uma. Introduza o mesmo endereço de e-mail em que recebeu o e-mail de convite no passo 6 acima.

    ![criar a conta Microsoft](media/speech-devices-sdk/get-sdk-3.png)

4. Clique em **seguinte** para criar uma palavra-passe.

5. Quando lhe for pedido para verificar seu email, regresse à sua caixa de entrada de e-mail para obter o código de verificação que é enviado para si.
 
7. Cole ou introduza o código de segurança da mensagem de e-mail na caixa de diálogo. Neste exemplo, é "8406." Clique depois em **Seguinte**.

    ![verificar e-mail](media/speech-devices-sdk/get-sdk-6.png)
 
8. Quando vir o aplicativo de painel de acesso na janela do browser, confirmar que o seu endereço de e-mail (no passo 6) agora faz parte do Azure Active Directory. Agora tem acesso ao site de download do SDK de dispositivos de voz.

## <a name="download-the-speech-devices-sdk"></a>Transferir os SDK de dispositivos de voz

Vá para o [site de download do SDK de dispositivos de voz](https://shares.datatransfer.microsoft.com/) e inicie sessão com a Microsoft Account que criou anteriormente. Agora pode transferir o SDK de dispositivos de voz, o código de exemplo associado e o material de referência ao seguir estes passos.

![site de download do SDK](media/speech-devices-sdk/get-sdk-7.png)

1. Baixe e instale a ferramenta Aspera ligar quando for solicitado a fazê-lo pelo navegador.

    ![Baixe o Aspera ligar](media/speech-devices-sdk/get-sdk-8.png)
 
1. Clique em **Sim** para mudar para ligar do Aspera.

    ![Mude para ligar do Aspera](media/speech-devices-sdk/get-sdk-9.png)
 
1. Clique em **permitir** para confirmar a transferir os ficheiros com o Aspera Connect.

    ![Transferir com Aspera Connect](media/speech-devices-sdk/get-sdk-10.png)
 
1. Feche a janela de transferências da ligação da Aspera depois dos ficheiros foram transferidos.

    ![Janela de transferências da ligação da Aspera](media/speech-devices-sdk/get-sdk-11.png)
 
Por predefinição, os ficheiros são transferidos para o seu **Downloads** pasta. Pode terminar sessão neste site agora.
