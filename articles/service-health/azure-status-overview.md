# <a name="azure-status"></a>Estado do Azure
[Estado do Azure](https://azure.microsoft.com/status/) fornece-lhe uma vista global do Estado de funcionamento de serviços do Azure e regiões. Com o estado do Azure, pode obter informações totalmente atualizadas sobre a disponibilidade de serviço. Estado do Azure está disponível para todos os utilizadores para ver todos os serviços que comunicam o estado de funcionamento do serviço, bem como incidentes com impacto abrangente. Se for um utilizador do Azure atual, no entanto, recomendamos que utilize a experiência personalizada no [do Azure Service Health](https://aka.ms/azureservicehealth). O Azure Service Health inclui todas as falhas, as atividades de manutenção planeada em breve e aconselhamentos sobre o serviço.

![Página de estado do Azure](./media/azure-status-overview/azure-status.PNG)

## <a name="azure-status-updates"></a>Atualizações de estado do Azure
A página de estado do Azure é atualizada em tempo real, como o estado de funcionamento de alteração de serviços do Azure. Se deixar a página de estado do Azure abertos, pode controlar a taxa a que a página é atualizada com novos dados. Na parte superior, pode ver a última vez que a página foi atualizada.

![Atualização do Estado do Azure](./media/azure-status-overview/update.PNG)

## <a name="azure-status-history"></a>Histórico do estado do Azure
Apesar da página de estado do Azure mostra sempre as últimas informações de estado de funcionamento, pode ver os eventos mais antigos com o [página de histórico do Estado do Azure](https://azure.microsoft.com/status/history/).

Como pode ver eventos de cópia de segurança para 90 dias no passado, incluindo uma causa raiz preliminar, atenuação e passos seguintes para o Azure.

## <a name="rss-feed"></a>RSS Feed
Estado do Azure também fornece [um RSS feed](https://azure.microsoft.com/status/feed/) das alterações para o estado de funcionamento dos serviços do Azure que pode subscrever.
