# <a name="mainframe-workloads-supported-on-azure"></a>Cargas de trabalho do mainframe suportadas no Azure

As soluções aqui listadas podem ajudá-lo a migrar a sua carga de trabalho para o Microsoft Azure. Algumas cargas de trabalho podem ser migradas com relativa facilidade. Outras cargas de trabalho que dependem do software do sistema legado podem ser realojadas. 

Para obter mais ajuda com a emulação e serviços do mainframe, consulte o centro de [migração Azure Mainframe](https://azure.microsoft.com/migration/mainframe/).

## <a name="migrate-mainframe-closer-to-azure"></a>Migrar mainframe mais perto de Azure

- [ASNA Visual RPG para .NET](https://asna.com/us/products/visual-rpg) compiler para o .NET Framework via Visual Studio plug-in.
- Ambiente de desenvolvimento [da ASYsco AMT COBOL](https://www.asysco.com/cobol/) (Unisys, mainframes IBM e outros dialetos COBOL, como micro Focus COBOL).
- Arquitetura de implantação baseada em nuvem [asysco AMT GO](https://www.asysco.com/amt-go/) para cargas de trabalho de alta qualidade.
- [Asysco AMT Transform](https://www.asysco.com/amt-transform/) para conversão de dados, código, scripting, segurança, interfaces e outros artefactos mainframe.
- [Fujitsu NetCOBOL](https://www.fujitsu.com/global/products/software/developer-tool/netcobol/) ferramentas de desenvolvimento e integração.
- Ferramentas de desenvolvimento e integração [micro Focus Visual COBOL.](https://www.microfocus.com/products/visual-cobol/)
- [Micro Focus PL/I](https://www.microfocus.com/campaign/download/pli-modernization/) compila legado para a plataforma .NET, suportando sintaxe PL/I mainframe, tipos de dados e comportamento.
- Plataforma de integração do [Micro Focus Enterprise Server.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)
- Ferramentas de desenvolvimento e integração da [MODERN Systems CTU (COBOL-To-Universal).](https://modernsystems.com/automatic-cobol-to-java-conversion/)
- [NTT Data Enterprise COBOL](https://us.nttdata.com/en/digital/application-development-and-modernization) ferramentas de desenvolvimento e integração.
- [NTT Open PL/I](https://us.nttdata.com/en/digital/application-development-and-modernization) compila para a plataforma .NET, suportando a sintaxe pl/I mainframe, tipos de dados e comportamento.
- Ferramentas de desenvolvimento e integração do [compilador COBOL](https://www.raincode.com/products/cobol/) de código de chuva.
- [O compilador de código de chuva PL/I](https://www.raincode.com/products/pli/) para a plataforma .NET suporta sintaxe PL/I, tipos de dados e comportamento.
- [Compilador de código de chuva ASM370](https://www.raincode.com/technical-landscape/asm370/) para o conjunto de computador principal 370 e sintaxe HLASM.

## <a name="deploy-an-emulation-environment-for-online-and-batch-processing"></a>Implementar um ambiente de emulação para processamento on-line e lote

- Arquitetura de implantação [asysco AMT GO](https://www.asysco.com/amt-go/) apoiando CICS, IMS, TIP, HVTIP e outros ambientes comuns do mainframe.
- Plataforma de integração do [Micro Focus Enterprise Server.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)
- [NTT DATA Mainframe Rehosting Development Environment](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf) ambiente de processamento de transações nativas.
- [NTT DATA Batch Processing Environment](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf) (BPE) incluindo capacidades de transação JCL.
- [Emulador CICS](https://www.raincode.com/technical-landscape/cics/) de código de chuva para plataformas .NET e Azure.
- Intérprete JCL compatível com plug-compatível com código [de chuva JCL.](https://www.raincode.com/products/jcl/)

## <a name="code-conversion"></a>Conversão de código

- Tecnologia de conversão do sistema [Asysco](https://www.asysco.com/azure-cloud/) que cobre código fonte, dados, lote, agendamento, monitores TP, interfaces, segurança, gestão e muito mais.
- [Serviços asysco AMT para](https://www.asysco.com/migration-services/) projetos de migração, incluindo inventário e análise, formação de design, ensaios de vestuário, go-live e apoio pós-migração.
- [Ferramentas Blu Age](https://www.bluage.com/) para digitalizar aplicações de negócios legados e bases de dados.
- [Serviços de Computação Heirloom](https://www.heirloomcomputing.com/tag/convert-cobol-to-java/) para converter o mainframe COBOL, CICS e VSAM para Java.
- [O LzLabs Software Defined Mainframe](https://www.lzlabs.com/) geriu o recipiente de software gerido para migrar aplicações mainframe para computadores Linux ou ambientes de nuvem privada, pública e híbrida.
