# <a name="deploy-a-kubernetes-workload-using-gpu-sharing-on-your-azure-stack-edge-pro"></a>Implemente uma carga de trabalho kubernetes usando a partilha de GPU no seu Azure Stack Edge Pro

Este artigo descreve como as cargas de trabalho contentorizadas podem partilhar as GPUs no seu dispositivo GPU Azure Stack Edge Pro. Neste artigo, você terá dois empregos, um sem a partilha de contexto da GPU e outro com a partilha de contexto ativada através do Serviço Multi-Processo (MPS) no dispositivo. Para mais informações, consulte o [Serviço Multi-Processo.](https://docs.nvidia.com/deploy/pdf/CUDA_Multi_Process_Service_Overview.pdf)

## <a name="prerequisites"></a>Pré-requisitos

Antes de começar, certifique-se de que:

1. Você tem acesso a um dispositivo GPU Azure Stack Edge Pro que é [ativado](azure-stack-edge-gpu-deploy-activate.md) e tem [computo configurado ](azure-stack-edge-gpu-deploy-configure-compute.md). Tem o ponto final da [API da Kubernetes](azure-stack-edge-gpu-deploy-configure-compute.md#get-kubernetes-endpoints) e adicionou este ponto final ao ficheiro do `hosts` seu cliente que irá aceder ao dispositivo.

1. Tem acesso a um sistema de clientes com um [sistema operativo suportado.](azure-stack-edge-gpu-system-requirements.md#supported-os-for-clients-connected-to-device) Se utilizar um cliente Windows, o sistema deverá executar o PowerShell 5.0 ou mais tarde para aceder ao dispositivo.

1. Criou um espaço de nome e um utilizador. Também concedeu ao utilizador o acesso a este espaço de nome. Tem o ficheiro kubeconfig deste espaço de nome instalado no sistema de clientes que utilizará para aceder ao seu dispositivo. Para obter instruções detalhadas, consulte [Connect to e gerencie um cluster Kubernetes através de kubectl no seu dispositivo GPU Azure Stack Edge Pro](azure-stack-edge-gpu-create-kubernetes-cluster.md#configure-cluster-access-via-kubernetes-rbac). 

1. Guarde a seguinte implantação `yaml` no seu sistema local. Usará este ficheiro para executar a implantação de Kubernetes. Esta implantação baseia-se em [recipientes Simples CUDA](https://docs.nvidia.com/cuda/wsl-user-guide/index.html#running-simple-containers) que estão disponíveis publicamente a partir da Nvidia. 

    ```yml
    apiVersion: batch/v1
    kind: Job
    metadata:
      name: cuda-sample1
    spec:
      template:
        spec:
          hostPID: true
          hostIPC: true
          containers:
            - name: cuda-sample-container1
              image: nvidia/samples:nbody
              command: ["/tmp/nbody"]
              args: ["-benchmark", "-i=1000"]
              env:
              - name: NVIDIA_VISIBLE_DEVICES
                value: "0"
          restartPolicy: "Never"
      backoffLimit: 1
    ---
    
    apiVersion: batch/v1
    kind: Job
    metadata:
      name: cuda-sample2
    spec:
      template:
        metadata:
        spec:
          hostPID: true
          hostIPC: true
          containers:
            - name: cuda-sample-container2
              image: nvidia/samples:nbody
              command: ["/tmp/nbody"]
              args: ["-benchmark", "-i=1000"]
              env:
              - name: NVIDIA_VISIBLE_DEVICES
                value: "0"
          restartPolicy: "Never"
      backoffLimit: 1
    ```

## <a name="job-without-context-sharing"></a>Trabalho sem partilha de contexto

Você executará o primeiro trabalho para implementar uma aplicação no seu dispositivo no espaço de `mynamesp1` nomes . Esta implementação de aplicações também mostrará que a partilha de contexto da GPU não é ativada por padrão. 

1. Lista todas as cápsulas em execução no espaço de nomes. Execute o seguinte comando: 

    ```powershell
    kubectl get pods -n <Name of the namespace>
    ```

    Aqui está uma saída de exemplo:

    ```powershell
    PS C:\WINDOWS\system32> kubectl get pods -n mynamesp1
    No resources found.
    ```
1. Inicie um trabalho de implantação no seu dispositivo utilizando o dispositivo de implementação.yaml fornecido anteriormente. Execute o seguinte comando: 

    ```powershell
    kubectl apply -f <Path to the deployment .yaml> -n <Name of the namespace> 
    ```  

    Este trabalho cria dois contentores e executa uma simulação n-body em ambos os contentores. O número de iterações de simulação é especificado no `.yaml` . Para obter mais informações, consulte [a simulação do corpo N.](https://physics.princeton.edu//~fpretori/Nbody/intro.htm)

## <a name="delete-deployment"></a>Eliminar implantação

Pode ser necessário eliminar as implementações quando estiver a funcionar com MPS ativado e com o MPS desativado no seu dispositivo.

Para eliminar a implantação do seu dispositivo, executar o seguinte comando: 

```powershell
kubectl delete -f <Path to the deployment .yaml> -n <Name of the namespace> 
```

Aqui está uma saída de exemplo:

```powershell
PS C:\WINDOWS\system32> kubectl delete -f 'C:\gpu-sharing\k8-gpusharing.yaml' -n mynamesp1
deployment.apps "cuda-sample1" deleted
deployment.apps "cuda-sample2" deleted
PS C:\WINDOWS\system32>
```
