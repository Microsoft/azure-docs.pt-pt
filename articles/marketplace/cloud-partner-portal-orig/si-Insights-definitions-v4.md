<a name="seller-insights-definitions"></a>Definições de insights de vendedor
=======================

O quadro seguinte fornece definições para muitos dos termos utilizados na Seller Insights.

|  **Termo**                 |  **Definição**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Tipo de licença Azure**                               | O tipo de contrato de licenciamento utilizado pelos clientes para a compra do Azure.  Também conhecido como Canal.                                                  |
| **Tipo de licença Azure: Fornecedor de soluções em nuvem**      | O cliente final adquire a oferta do Azure e do seu Marketplace através do seu Fornecedor de Soluções Cloud, que atua como o seu revendedor.                 |
| **Tipo de licença Azure: Enterprise**                   | O cliente final adquire a Oferta Azure e o seu Marketplace através de um Acordo de Empresa, assinado diretamente com a Microsoft.                  |
| **Tipo de licença Azure: Empresa através de Revendedor**  | O cliente final adquire a oferta do Azure e do seu Marketplace através de um revendedor que facilita o seu Acordo de Empresa com a Microsoft.     |
| **Tipo de licença Azure: Pague à medida que vai**                | O cliente final adquire a oferta do Azure e do seu Marketplace através de um acordo "Pay as You Go", assinado diretamente com a Microsoft.                |
| **Valor de Cobrança (CC)**                              | O valor cobrado ao cliente, na *Moeda de Cliente* (CC) para faturação.                                                                 |
| **Valor de Cobrança (PC)**                               | O valor cobrado ao cliente, na Moeda de *Pagamento* (PC).                                                                      |
| **Data de Carregamento**                                      | A data em que o custo do cliente foi calculado (normalmente imediatamente após o período de utilização).                                             |
| **Nome de instância de nuvem**                              | O Microsoft Cloud em que ocorreu uma implementação vm.                                   |
| **Cloud Instance Name: Azure Global**                | A nuvem pública global da Microsoft.                           |
| **Cloud Instance Name: Governo Azure**        | Nuvens específicas do governo para um dos seguintes governos: China, Alemanha ou Estados Unidos da América.                           |
| **Estado de Recolha**  | Por uma taxa específica, o mais recente estado do ciclo de vida de faturação e recolha.  Por exemplo, o estado poderia ser recolhido, recolhido ou reembolsado.                  |
| **Tamanho do núcleo**  | O número de núcleos virtuais apresentados pela máquina virtual a oferta está a decorrer.               |
| **Cidade do Cliente**  | O nome da cidade fornecido pelo cliente. Isto pode ser diferente da cidade na assinatura Azure de um cliente.  |
| **Linguagem de comunicação do cliente**        | A linguagem preferida pelo cliente para a comunicação.                                  |
| **Nome da empresa cliente**                  | O nome da empresa fornecido pelo cliente. Isto pode ser diferente da cidade na assinatura Azure de um cliente.                                  |
| **País cliente**                       | O nome país/região fornecido pelo cliente.  Isto poderia ser diferente do país/região na Subscrição Azure de um cliente.                               |
| **Moeda do Cliente (CC)**                 | A moeda preferida pelo cliente para preços e faturação.                            |
| **E-mail do cliente**                         | O endereço de e-mail fornecido pelo cliente final.  Isto pode ser diferente do endereço de e-mail na Subscrição Azure de um cliente.                   |
| **Primeiro nome do cliente**                              | O primeiro nome fornecido pelo cliente.  Isto pode ser diferente do nome fornecido na Subscrição Azure de um cliente.                    |
| **Nome do cliente**                               | O último nome fornecido pelo cliente.  Isto pode ser diferente do nome fornecido na Subscrição Azure de um cliente.                    |
| **ID de Cliente**                               | O identificador único atribuído a um cliente.  Um cliente pode ter zero ou mais subscrições do Azure Marketplace.                    |
| **Tipo de pagamento do cliente**                            | O tipo de instrumento de pagamento utilizado pelo cliente.  Por exemplo, este pode ser cartão, fatura ou cheque eletrônico.                             |
| **Código Postal do Cliente**                             | O código postal fornecido pelo cliente.  Isto poderia ser diferente do código postal fornecido na Assinatura Azure de um cliente.                                                                 |
| **Estado cliente**                                   | O estado (endereço) fornecido pelo cliente.  Isto pode ser diferente do último nome fornecido na Subscrição Azure de um cliente.                                                                 |
| **Data Adquirida**                                    | A primeira data em que o cliente adquiriu qualquer oferta publicada por si.                 |
| **Data Perdida**                                        | A última data em que o cliente cancelou a última de todas as ofertas anteriormente adquiridas.                                                         |
| **Encargos estimados alargados (CC)**                   | A taxa estimada e alargada para a quantidade de unidades de utilização para um dado SKU (na moeda do cliente). Este valor pode não ser exato devido a erros de arredondamento ou truncação.                            |
| **Carga alargada estimada (PC)**                   | Calcula-se a taxa alargada estimada para a quantidade de unidades de utilização de um dado SKU, com base na conversão cambial da data de utilização (na moeda do editor). Este valor pode não ser exato devido a erros de arredondamento ou truncação.                  |
| **Pagamento Estimado (PC)**                            | O pagamento estimado pela quantidade de unidades de utilização de um dado SKU, com base na conversão cambial na data em que a utilização é calculada (na moeda do editor). Este valor pode não ser exato devido a erros de arredondamento ou truncação.                        |
| **Preço Estimado (PC)**                             | Calcula-se o preço estimado para uma unidade de utilização de um dado SKU, com base na conversão cambial da data de utilização (na moeda do editor). Este valor pode não ser exato devido a erros de arredondamento ou truncação.                                             |
| **É novo cliente**                                  | O valor irá informá-lo se é um novo cliente consumindo ou comprando uma ou mais ofertas do seu Marketplace pela primeira vez (ou não).  O valor será "Sim" se for o mesmo mês civil para "Data Adquirida".  O valor será "Não" se o cliente tiver adquirido alguma das suas ofertas antes do mês civil reportado.                                                       |
| **É Pré-visualização SKU**                                  | O valor irá informá-lo se marcou o SKU como "pré-visualização". O valor será "Sim" se o SKU tiver sido marcado em conformidade, e apenas as subscrições do Azure autorizadas por si podem implementar e utilizar esta imagem.  O valor será "Não" se o SKU não tiver sido identificado como "pré-visualização".                                          |
| **É o Contacto Promocional Opt in**                   | O valor irá informá-lo se o cliente optou proativamente pelo contacto promocional da editora. Neste momento, não estamos a apresentar a opção aos clientes, pelo que indicámos "Não" em todo o tabuleiro. Assim que esta funcionalidade for implementada, começaremos a atualizar em conformidade.                                          |
| **Tipo de Licença de Mercado**                         | O método de faturação da oferta do Marketplace.                             |
| **Tipo de licença de mercado: Faturado através do Azure**   | A Microsoft é o seu agente para esta oferta do Marketplace e fatura clientes em seu nome. (Cartão de crédito PAYG ou Fatura da Empresa)       |
| **Tipo de licença de mercado: traga a sua própria licença** | O VM requer uma chave de licença fornecida pelo cliente para implementar. A Microsoft não fatura clientes pelo uso desta oferta do Marketplace. |
| **Tipo de licença de mercado: Grátis**                   | A oferta do Marketplace está configurada para ser gratuita para todos os utilizadores. A Microsoft não fatura clientes pelo uso desta oferta do Marketplace.    |
| **Tipo de licença de mercado: Microsoft como Revendedor**  | A Microsoft é o seu revendedor para esta oferta do Marketplace.                               |
| **Nome da empresa de subscrição do mercado**  | O nome da empresa fornecido pelo cliente na sua Assinatura Azure.                           |
| **ID de subscrição do mercado**            | O identificador único associado à Subscrição Azure o cliente usou para comprar a sua oferta de Marketplace.  Antigamente era o GUIA de Subscrição Azure.  Em alguns casos, como uma encomenda SAAS, não é necessária nenhuma subscrição.           |
| **Taxa Microsoft (CC)**                               | A taxa da Microsoft sobre a transação na moeda do cliente.                              |
| **Nome da oferta**                                | O nome da oferta do Marketplace.                                                             |
| **Tipo de Oferta**                                | O tipo de oferta do Microsoft Marketplace.  Por exemplo, poderia ser gerida aplicação, licenças de máquinavirtual ou contentor.                           |
| **Data de cancelamento de encomenda**                         | A data em que a encomenda do Marketplace foi cancelada.                                                       |
| **ID da Encomenda**                                  | O identificador único da encomenda do cliente para o seu serviço Marketplace.  As ofertas baseadas na utilização da Máquina Virtual não estão associadas a uma encomenda.                 |
| **Data de Compra de Encomendas**                       | A data em que a Ordem do Mercado foi criada.                                                        |
| **Quantidade de Encomendas**                       | A quantidade de encomendas para o mesmo SKU associado a uma Subscrição Azure.  Para as encomendas da SAAS, representa o número de lugares associados a este caso de encomenda.                                            |
| **Estado da Encomenda**                              | O estado de uma encomenda do Marketplace na altura em que os dados foram atualizados pela última vez.  Por exemplo, a encomenda pode estar ativa ou cancelada.             |
| **Estado da encomenda: Ativo**                           | O cliente comprou uma encomenda e não cancelou a encomenda.   |
| **Estado da encomenda: Cancelado**                        | O cliente previamente adquiriu uma encomenda e posteriormente cancelou a sua encomenda.        |
| **Valor de Pagamento (PC)**                             | O valor que lhe foi pago, na sua moeda de pagamento preferida (PC).                                       |
| **Moeda de pagamento (PC)**                           | A moeda usada para os seus pagamentos.                                                                |
| **Data de Pagamento**                                    | A data em que o pedido de pagamento foi enviado da Microsoft para a sua instituição financeira escolhida.                                                                         |
| **Estado do pagamento**                                  | O estado do pagamento no momento em que os dados foram atualizados pela última vez.                                   |
| **Estado do pagamento: Em curso**               | A transação não está pronta para ser paga. (Para mais informações, consulte o Estado da Recolha)                             |
| **Estado do pagamento: Pago**                          | A transação foi incluída num cálculo de pagamento passado. Os valores positivos são pagos e os valores negativos são obtidos contra o montante total devido. |
| **Estado do pagamento: Próximo pagamento**                   | A transação está pronta para ser paga e será incluída no próximo cálculo de pagamento disponível.                      |
| **Preço (CC)**                                   | O preço de uma unidade de utilização para um dado SKU (na moeda do cliente).                                                                  |
| **Email do fornecedor**                                 | O endereço de e-mail do fornecedor envolvido na relação entre a Microsoft e o cliente final. Se o cliente for uma Enterprise através do Revendedor, será o revendedor.  Se estiver envolvido um Fornecedor de Soluções cloud (CSP), será o CSP.                                                                 |
| **Nome do fornecedor**                                 | O nome do fornecedor envolvido na relação entre a Microsoft e o cliente final. Se o cliente for uma Enterprise através do Revendedor, será o revendedor.  Se estiver envolvido um Fornecedor de Soluções cloud (CSP), será o CSP.                                                                 |
| **Recurso URI**                                  | O caminho onde o artefacto de implantação foi implantado.                                            |
| **SKU**                                           | Nome SKU definido durante a publicação. Uma oferta pode ter muitos SKUs, mas um SKU só pode ser associado a uma única oferta.              |
| **Tipo de faturação SKU**                                 | O método de faturação do SKU.                                                                                                             |
| **Tipo de faturação SKU: Traga a sua própria licença**         | O VM ou serviço requer uma chave de licença fornecida pelo cliente para implementar. A Microsoft não fatura clientes pelo uso destas ofertas do Marketplace.   |
| **Tipo de faturação SKU: Microsoft Como Revendedor**          | A Microsoft é o seu revendedor para este SKU.   |
| **Tipo de faturação SKU: Grátis**                           | O SKU está configurado para ser gratuito para todos os utilizadores. A Microsoft não fatura clientes pelo uso deste SKU.                           |
| **Tipo de faturação SKU: Pago**                           | A Microsoft é o seu agente para este SKU e fatura clientes em seu nome. (Cartão de crédito PAYG ou Fatura da Empresa)                   |
| **Tipo de faturação SKU: Ensaio**                          | O cliente encontra-se em período experimental e será convertido para pagamento se não cancelar ou apagar.                                         |
| **Valor Fiscal (CC)**                                  | O valor do imposto aplicado à conta do cliente na Moeda do Cliente (CC).                                                               |
| **Data de Transação**                                 | A data da transação registada no seu relatório de pagamento.                                                                           |
| **Tipo de Transação**                               | O tipo de transação que está a ser reportada.  Por exemplo, este pode ser um ajuste de custos, reembolso ou pagamento.                                                                                        |
| **Data final do julgamento**                                 | A data em que o período experimental desta ordem terminará ou terminou.                                                                           |
| **Data de Utilização**                                       | A data de utilização do cliente ocorreu.                                                                                                             |
| **Quantidade de utilização**                                       | O uso reportado do cliente do SKU. Para imagens VM, os registos de utilização representam o uso para o período reportado para o tamanho vm e SKU.                                               |
| **Referência de utilização**                                       | O identificador para um ou mais dias de utilização do cliente para um dado SKU associado a uma entrada no relatório de pagamento.                                               |
| **Tipo de Utilização**                                       | Uma descrição do uso que está a ser medido. (Utilização normalizada ou utilização bruta)                                                              |
| **Tipo de utilização: Utilização normalizada**                | As horas de utilização normalizadas para contabilizar o número de núcleos VM envolvidos na utilização: [Número de Núcleos VM] x [Horas de Utilização Bruta]. VMs designados como "SHAREDCORE" utilizam 1/6 (ou 0,1666) como multiplicador [Número de Núcleos VM].                                        |
| **Tipo de utilização: Uso bruto**                        | Horas de utilização para a sua oferta do Marketplace.  É apresentado em termos de horário de "calendário".  É o número de máquinas virtuais de tempo que têm funcionado.                           |
| **Unidades de Utilização**                                   | O aparelho de medição para a utilização indicada. Os VMs são sempre medidos com unidades de medição horárias.                               |
| **Tamanho VM**                                       | Representa o tamanho do hardware da máquina virtual alinhado com a oferta Azure. Exemplos `Basic_A0` `Standard_A11`incluem, , , `Standard_D12`e `Standard_G4`.   |
|  |  |
