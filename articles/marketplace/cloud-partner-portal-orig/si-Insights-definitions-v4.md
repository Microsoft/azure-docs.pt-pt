<a name="seller-insights-definitions"></a>Definições de Insights do vendedor
=======================

A tabela seguinte fornece definições para muitos dos termos utilizados no Insights de vendedor.

|  **Termo**                 |  **Definição**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Tipo de licença do Azure**                               | O tipo de contrato de licenciamento utilizado pelos clientes para comprar o Azure.  Também conhecido como canal.                                                  |
| **Tipo de licença do Azure: Fornecedor de soluções cloud**      | O cliente final procures do Azure e a sua oferta do Marketplace através do respetivo fornecedor de soluções Cloud, que atua como seu revendedor.                 |
| **Tipo de licença do Azure: Enterprise**                   | O cliente final procures do Azure e a sua oferta do Marketplace por meio de um Enterprise Agreement, assinado diretamente com a Microsoft.                  |
| **Tipo de licença do Azure: Empresa por meio do revendedor**  | O cliente final procures do Azure e a sua oferta do Marketplace por meio de um revendedor que facilita o contrato Enterprise com a Microsoft.     |
| **Tipo de licença do Azure: Pay as You Go**                | O cliente final procures do Azure e a sua oferta do Marketplace por meio de um contrato de "Pay as You Go", iniciada diretamente com a Microsoft.                |
| **Quantidade de custos (CC)**                              | O valor cobrado ao cliente, o *moeda de cliente* (CC) para faturação.                                                                 |
| **Quantidade de custos (PC)**                               | O valor cobrado ao cliente, o *dividendos moeda* (PC).                                                                      |
| **Data de cobrança**                                      | A data de que cobrança do cliente foi calculada (normalmente, imediatamente após o período de utilização).                                             |
| **Nome da instância de cloud**                              | A Cloud da Microsoft em que ocorreu uma implementação de VM.                                   |
| **Nome da instância de cloud: Azure Global**                | A nuvem pública da Microsoft global.                           |
| **Nome da instância de cloud: Azure Government**        | Nuvens de Microsoft específico da administração pública para um dos Governos seguintes: China, Alemanha ou dos Estados Unidos da América.                           |
| **Estado da coleção**  | Por um valor específico, o estado mais recente do ciclo de vida de faturação e de coleção.  Por exemplo, o estado pode ser a coleção em curso, recolhidos, ou o reembolso.                  |
| **Tamanho de núcleo**  | O número de núcleos virtuais apresentado pela máquina virtual que a oferta está a ser executada.               |
| **Cidade do cliente**  | O nome de cidade fornecido pelo cliente. Isto pode ser diferente de cidade na subscrição do Azure de um cliente.  |
| **Idioma de comunicação de cliente**        | O idioma preferido pelo cliente para comunicação.                                  |
| **Nome da empresa do cliente**                  | O nome da empresa fornecido pelo cliente. Isto pode ser diferente de cidade na subscrição do Azure de um cliente.                                  |
| **País do cliente**                       | O nome do país fornecido pelo cliente.  Isto pode ser diferente do país na subscrição do Azure de um cliente.                               |
| **Moeda de cliente (CC)**                 | A moeda preferida pelo cliente para preços e faturação.                            |
| **Cliente de E-Mail**                         | O endereço de email fornecido pelo cliente final.  Isto pode ser diferente da subscrição do Azure de um cliente, o endereço de email.                   |
| **Nome do primeiro cliente**                              | O primeiro nome fornecido pelo cliente.  Isto pode ser diferente do nome fornecido na subscrição do Azure de um cliente.                    |
| **Apelido do cliente**                               | O último nome fornecido pelo cliente.  Isto pode ser diferente do nome fornecido na subscrição do Azure de um cliente.                    |
| **ID de cliente**                               | O identificador exclusivo atribuído a um cliente.  Um cliente pode ter zero ou mais subscrições de mercado do Azure.                    |
| **Tipo de pagamento de cliente**                            | O tipo de instrumento de pagamento utilizado pelo cliente.  Por exemplo, isso pode ser cartão, nota fiscal ou verificação de eletrônica.                             |
| **Código Postal do cliente**                             | O código postal fornecido pelo cliente.  Isto pode ser diferente do CEP fornecido na subscrição do Azure de um cliente.                                                                 |
| **Estado do cliente**                                   | O estado (endereço) fornecido pelo cliente.  Isto pode ser diferente do último nome fornecido na subscrição do Azure de um cliente.                                                                 |
| **Data adquirida**                                    | A data do primeiro o cliente comprou qualquer oferta publicada por si.                 |
| **Data perdida**                                        | A data da última o cliente foi cancelada a última das todas as ofertas anteriormente comprada.                                                         |
| **Custo estimado de expandida (CC)**                   | O estimado estendido custos para a quantidade de unidades de utilização para um determinado SKU (na moeda do cliente). Este valor pode não ser exato devido a erros de arredondamento ou o truncamento.                            |
| **Custo estimado de expandida (PC)**                   | Expandido custos para a quantidade de unidades de utilização para um determinado SKU, com base na conversão de câmbio na utilização da data estimada é calculada (na moeda do Editor). Este valor pode não ser exato devido a erros de arredondamento ou o truncamento.                  |
| **Dividendos estimado (PC)**                            | O estimado pagamento para a quantidade de unidades de utilização para um determinado SKU, com base na conversão de câmbio na data que da utilização é calculada (na moeda do Editor). Este valor pode não ser exato devido a erros de arredondamento ou o truncamento.                        |
| **Preço estimado (PC)**                             | O preço estimado para uma unidade de utilização para um determinado SKU, com base na conversão de câmbio sobre a utilização de data é calculado (na moeda do Editor). Este valor pode não ser exato devido a erros de arredondamento ou o truncamento.                                             |
| **É o novo cliente**                                  | O valor irá informá-lo se é um novo cliente a consumir ou comprar um ou mais dos seus Marketplace oferece pela primeira vez (ou não).  O valor será "Sim" se for o mesmo mês de calendário para "Adquiriu a data".  O valor será "Não", se o cliente tenha adquirido qualquer uma das suas ofertas antes do mês de calendário comunicado.                                                       |
| **É a pré-visualização do SKU**                                  | O valor irá informá-lo a se marcaram o SKU como "pré-visualização". O valor será "Sim" se o SKU tenha sido identificado em conformidade e as subscrições apenas do Azure autorizadas por si, podem implementar e utilizar esta imagem.  O valor será "Não", se o SKU não foi identificado como "pré-visualização".                                          |
| **Contacto promocional é escolher**                   | O valor irá informá-lo a se o cliente proativamente selecionada para promocional, contacte de publicadores. Neste momento, nós não a apresentação a opção aos clientes, para que tenha indicado "Não", em geral. Assim que esta funcionalidade é implementada, começaremos a atualização da mesma forma.                                          |
| **Tipo de licença do Marketplace**                         | O método de faturação da oferta do Marketplace.                             |
| **Tipo de licença do Marketplace: Cobrado por meio do Azure**   | A Microsoft é o seu agente para esta oferta do Marketplace e cobra aos clientes em seu nome. (PAYG cartão de crédito ou fatura Enterprise)       |
| **Tipo de licença do Marketplace: Traga a sua própria licença** | A VM requer uma chave de licença fornecida pelo cliente para implementar. Microsoft não cobra os clientes para a utilização deste ofertas do Marketplace. |
| **Tipo de licença do Marketplace: Gratuito**                   | A oferta do Marketplace está configurada para ser livre para todos os utilizadores. A Microsoft não cobrar aos clientes para a utilização desta oferta do Marketplace.    |
| **Tipo de licença do Marketplace: Microsoft as Reseller**  | A Microsoft é o seu revendedor para esta oferta do Marketplace.                               |
| **Nome da empresa de subscrição do Marketplace**  | O nome da empresa fornecido pelo cliente na respetiva subscrição do Azure.                           |
| **ID de subscrição do Marketplace**            | O identificador exclusivo associado, com a subscrição do Azure, o cliente utilizado para comprar a sua oferta do Marketplace.  Anteriormente, era o GUID de subscrição do Azure.  Em alguns casos, como uma ordem SAAS, não é necessária nenhuma subscrição.           |
| **Taxa da Microsoft (CC)**                               | A taxa de Microsoft na transação na moeda do cliente.                              |
| **Nome da oferta**                                | O nome da oferta do Marketplace.                                                             |
| **Tipo de oferta**                                | O tipo de oferta do Microsoft Marketplace.  Por exemplo, é possível aplicação gerida, licenças de máquina virtual ou contentor.                           |
| **Data de cancelamento do pedido**                         | A data em que a ordem de Marketplace foi cancelada.                                                       |
| **ID da encomenda**                                  | O identificador exclusivo do pedido de cliente para o seu serviço do Marketplace.  Ofertas de baseada na utilização da máquina virtual não estão associadas um pedido.                 |
| **Data de compra do pedido**                       | A data em que a ordem de Marketplace foi criada.                                                        |
| **Quantidade de pedidos**                       | A quantidade de pedidos para o mesmo SKU associado a uma subscrição do Azure.  Para pedidos SAAS, ele representa o número de utilizadores associados a esta instância de um pedido.                                            |
| **Estado da encomenda**                              | O estado de uma ordem de Marketplace ao tempo que os dados foi atualizados pela última vez.  Por exemplo, a ordem pode ser ativo ou foi cancelada.             |
| **Estado da encomenda: Active**                           | O cliente tenha adquirido uma encomenda e não cancelou a sua ordem.   |
| **Estado da encomenda: Foi cancelada**                        | O cliente comprou uma encomenda e, em seguida, cancelada sua ordem.        |
| **Quantidade de dividendos (PC)**                             | O valor pago para, na moeda dividendos preferencial (PC).                                       |
| **Moeda de dividendos (PC)**                           | A moeda utilizada para seu dividendos.                                                                |
| **Data de dividendos**                                    | A data do pedido de pagamento foi enviado da Microsoft a sua instituição financeira escolhida.                                                                         |
| **Estado de dividendos**                                  | O estado de dividendos ao tempo que os dados foi atualizados pela última vez.                                   |
| **Estado de dividendos: Em curso**               | A transação não está pronta para dividendos. (Para obter mais informações, veja o estado de recolha)                             |
| **Estado de dividendos: Pago**                          | A transação foi incluída num cálculo dividendos anteriores. Valores positivos são pagas, e valores negativos são netted em relação a quantia total devida. |
| **Estado de dividendos: Dividendos futuros**                   | A transação está pronta para dividendos e será incluída no cálculo dividendos disponível seguinte.                      |
| **Preço (CC)**                                   | O preço de uma unidade de utilização para um determinado SKU (na moeda do cliente).                                                                  |
| **Fornecedor de E-Mail**                                 | O endereço de e-mail do fornecedor envolvido na relação entre a Microsoft e o cliente final. Se o cliente é uma empresa por meio de revendedor, será o revendedor.  Se um fornecedor de soluções Cloud (CSP) estiver envolvido, é o CSP.                                                                 |
| **Nome do fornecedor**                                 | O nome do fornecedor envolvido na relação entre o cliente da Microsoft e de fim. Se o cliente é uma empresa por meio de revendedor, será o revendedor.  Se um fornecedor de soluções Cloud (CSP) estiver envolvido, é o CSP.                                                                 |
| **URI do recurso**                                  | O caminho onde o artefacto de implantação foi implementado.                                            |
| **SKU**                                           | Nome SKU conforme definido durante a publicação. Uma oferta pode ter várias SKUs, mas um SKU só pode ser associado com uma única oferta.              |
| **Tipo de faturação de SKU**                                 | O método de faturação do SKU.                                                                                                             |
| **Tipo de faturação de SKU: Traga a sua própria licença**         | A VM ou serviço requer uma chave de licença fornecida pelo cliente para implementar. A Microsoft não cobrar aos clientes para a utilização destas ofertas do Marketplace.   |
| **Tipo de faturação de SKU: Microsoft As Reseller**          | A Microsoft é o seu revendedor para este SKU.   |
| **Tipo de faturação de SKU: Gratuito**                           | O SKU está configurado para ser livre para todos os utilizadores. A Microsoft não cobrar os clientes para a utilização deste SKU.                           |
| **Tipo de faturação de SKU: Pago**                           | A Microsoft é o seu agente para este SKU e cobra aos clientes em seu nome. (PAYG cartão de crédito ou fatura Enterprise)                   |
| **Tipo de faturação de SKU: Trial**                          | O cliente está no seu período de avaliação e será convertido para pago se eles não cancelar ou eliminar.                                         |
| **Valor do imposto (CC)**                                  | O valor do imposto aplicado à fatura do cliente no cliente moeda (CC).                                                               |
| **Data da transação**                                 | A data da transação registadas no seu relatório de dividendos.                                                                           |
| **Tipo de transação**                               | O tipo de transação que está a ser comunicado.  Por exemplo, isso pode ser um custo, reembolso ou ajuste dividendos.                                                                                        |
| **Data de fim de avaliação**                                 | A data do período de avaliação para este pedido vai terminar ou terminou.                                                                           |
| **Data de utilização**                                       | Ocorreu a utilização do cliente de data.                                                                                                             |
| **Quantidade de utilização**                                       | A utilização de relatados pelos clientes do SKU. Para imagens de VM, registos de utilização representam a utilização para o período de comunicados para esse SKU e o tamanho da VM.                                               |
| **Referência de utilização**                                       | O identificador para um ou mais dias de utilização do cliente para um determinado SKU associado com uma entrada no relatório dividendos.                                               |
| **Tipo de utilização**                                       | Uma descrição da utilização da que está a ser medida. (Utilização normalizada ou utilização não processada)                                                              |
| **Tipo de utilização: Utilização normalizada**                | Horas de utilização normalizados para a conta para o número de núcleos VM envolvidos na utilização: [número de núcleos de VM] x [horas de utilização não processados]. VMs indicadas como "SHAREDCORE" utiliza 1/6 (ou 0.1666) como o multiplicador de [número de núcleos de VM].                                        |
| **Tipo de utilização: Utilização não processada**                        | Horas de utilização para a sua oferta do Marketplace.  Ele é apresentado em termos de horas de "calendário".  É o número de máquinas de virtuais de tempo tem de estar em execução.                           |
| **Unidades de utilização**                                   | A unidade de medida para o uso declarado. As VMs são sempre medidas com unidades por hora de medida.                               |
| **Tamanho da VM**                                       | Representa o tamanho de hardware da máquina virtual alinhado com a oferta do Azure. Os exemplos incluem `Basic_A0`, `Standard_A11`, `Standard_D12`, e `Standard_G4`.   |
|  |  |
