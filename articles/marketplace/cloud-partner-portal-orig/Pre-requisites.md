# <a name="prerequisites-for-application-curation"></a>Pré-requisitos para a organização de aplicação

A Microsoft requer revisões específicas para validar que são cumpridos os seguintes requisitos:

- Um parceiro\'código personalizado de s cumpre os diretrizes da Microsoft.
- Um pacote de solução do Microsoft Dynamics Lifecycle Services (LCS) pode ser implementado com êxito.
- Transações podem ser concluídas.

Atualmente, parceiros devem demonstrar que esses requisitos foram cumpridos por fazer as implementações de teste e, em seguida, partilhar os resultados com a Microsoft. Nenhum código será implantado num ambiente de cliente que cujos Microsoft\'t validada. Parceiros tem de concluir os seguintes artefatos de organização e os testes:

- Relatório de análise de código (CAR)
- O Modelador de processo de negócio (BPM) / scripts de teste
- Nome do projeto e uma descrição
- Cópia de segurança de base de dados de negócio
- Pacotes de dados e pacotes de dados do processo (PDPs)
- Metodologia
- Binários (opcionais)
- Pacotes de implantação possível
- Modelos (código e testes)
- Conteúdo de marketing

Para obter mais informações sobre os requisitos técnicos e de processos, consulte [validar aplicativos para finanças e operações](https://docs.microsoft.com/dynamics365/operations/dev-itpro/lcs-solutions/app-validation-lcs-solutions).  Além disso, se ainda não o fez, reveja os [Office 365, Dynamics 365, PowerApps e o Power BI oferecem guia de publicação](../appsource-offer-publishing-guide.md).
