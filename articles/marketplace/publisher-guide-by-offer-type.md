# <a name="publishing-guide-by-offer-type"></a>Guia de publicação por tipo de oferta

Assim que [decidir uma opção](https://docs.microsoft.com/azure/marketplace/determine-your-listing-type)de publicação, está pronto para selecionar o tipo de oferta que será utilizado para apresentar a sua oferta. 

 O tipo de *oferta* define a estrutura da oferta, que inclui os metadados, artefactos e outros conteúdos utilizados para apresentar a oferta no mercado.

Antes de poder criar uma oferta, tem de escolher um tipo de oferta. O tipo de oferta corresponderá ao tipo de solução, app ou oferta de serviço que pretende publicar, bem como ao seu alinhamento com os produtos e serviços da Microsoft. 

Um único tipo de oferta pode ser configurado de forma diferente para permitir diferentes opções de publicação, chamadas à ação, provisionamento ou preços. A opção de publicação e configuração do tipo de oferta também se alinham com a elegibilidade da oferta e requisitos técnicos. 

Certifique-se de rever a montra e oferecer requisitos de elegibilidade do tipo e os requisitos de publicação técnica antes de criar a sua oferta.

## <a name="list-of-offer-types"></a>Lista de tipos de oferta

Os tipos de oferta do Azure Marketplace estão listados na tabela abaixo.

| **Tipo de oferta**    | **Descrição**  |
| :------------------- | :-------------------|
| [**Virtual Machines**](https://docs.microsoft.com/azure/marketplace/marketplace-virtual-machines) | Utilize o tipo de oferta de máquina virtual quando colocar um aparelho virtual na subscrição associada ao seu cliente. |
| [**Modelos de solução**](https://docs.microsoft.com/azure/marketplace/marketplace-solution-templates) | Utilize o modelo de solução (também conhecido como aplicação Azure) quando a sua solução requer uma implementação adicional e automatização de configuração para além de um VM simples. Os modelos de solução podem empregar muitos tipos diferentes de recursos Azure, incluindo, mas não limitados a VMs.  |
| [**Aplicações geridas**](https://docs.microsoft.com/azure/marketplace/marketplace-managed-apps) | Utilize a aplicação Azure: tipo de oferta de aplicações gerida quando forem necessárias as seguintes condições: <br> <ul> <li> Implementa uma solução baseada em subscrição para o seu cliente utilizando um VM ou uma solução inteira baseada em IaaS. </li> <li>Você ou o seu cliente exigem que a solução seja gerida por um parceiro. </li> <ul> |
| [**Aplicações SaaS**](https://docs.microsoft.com/azure/marketplace/marketplace-saas-applications-technical-publishing-guide) | Utilize o tipo de oferta de aplicações SaaS para permitir ao seu cliente comprar a sua solução técnica baseada no SaaS como subscrição. |
| [**Ofertas de contentores**](https://docs.microsoft.com/azure/marketplace/marketplace-containers) | Utilize o tipo de oferta do Recipiente quando a sua solução for uma imagem de contentor Docker aprovisionada como um serviço de contentores Azure baseado em Kubernetes. |
| [**Módulos IoT Microsoft Edge**](https://docs.microsoft.com/azure/marketplace/iot-edge-module) | Os módulos Azure IoT Edge são as unidades de computação mais pequenas geridas pelo IoT Edge, e podem conter serviços da Microsoft (como o Azure Stream Analytics), serviços de terceiros ou o seu próprio código específico de solução. |
| [**Serviços de consultoria**](https://docs.microsoft.com/azure/marketplace/consulting-services) | Os Serviços de Consultoria ajudam a conectar os clientes com serviços de apoio e alargar a sua utilização dos serviços Azure, Dynamics 365 ou Power Suite.|


Por fim, consulte [**os requisitos da AD Azur,**](https://docs.microsoft.com/azure/marketplace/enable-appsource-marketplace-using-azure-ad) listando opções e oferecendo tipos para obter mais informações sobre os requisitos de inscrição simples.
