# <a name="power-bi-application-offer"></a>Oferta do Power BI aplicação

|                  |                                |
|------------------|--------------------------------|
| Esta secção explica como publicar uma aplicação do Power BI da Microsoft [Marketplace do AppSource](https://appsource.microsoft.com/). Uma aplicação do Power BI pacotes de conteúdos do Power BI e utiliza fluxos de dados para ligar relatórios e dashboards a dados no armazenamento de dados comuns. A aplicação pode ser implementada, em seguida, para outros inquilinos através do AppSource. | ![Ícone do Power BI](./media/powerbi-icon.png) |


Esta secção é dividida nessas partes principais:

-   [Pré-requisitos](./cpp-prerequisites.md) -oferecem os requisitos técnicos e empresariais para criar ou publicar uma aplicação do Power BI
-   [Criar oferta de aplicação do Power BI](./cpp-create-offer.md) -os passos necessários para criar uma nova aplicação do Power BI oferecem ao utilizar a entrada a [Cloud Partner Portal](https://cloudpartner.azure.com)
-   [Publicar oferta de aplicação do Power BI](./cpp-publish-offer.md) -como submeter uma nova oferta para publicação no AppSource e como atualizar uma oferta existente.


## <a name="publishing-steps"></a>Passos de publicação

O diagrama seguinte ilustra as etapas de alto nível em publicar uma oferta de aplicação do Power BI.

![Passos de publicação do Power BI aplicação](media/publishing-steps.png)

Processo de publicação do Power BI aplicações

1. Crie uma aplicação de serviço no Power BI (o que resulta na criação do URL do pacote de instalação).  Este URL representa os recursos técnicos para a oferta.  Para obter mais informações, consulte [o que é o Power BI](https://go.microsoft.com/fwlink/?linkid=2028636).
2. Recolher ou criar os ativos empresariais, incluindo:
    - Materiais de marketing
    - Legal e documentos de suporte
3.  Criar a oferta – utilizar o Portal de parceiros da Cloud para configurar os detalhes e informações sobre a oferta, incluindo a descrição da oferta, materiais, legal, as informações de suporte e as especificações dos ativos de marketing.
4.  Certificar e publicar a oferta - esta submissão inicia o processo de publicação.
    - A equipa de integração do AppSource testa, valida e certifica que seu aplicativo. 
    - Após a certificação, deve revê-lo no seu ambiente de teste e liberá-lo.

Depois de todas essas etapas são realizadas, a aplicação "passa em direto" no AppSource.

## <a name="next-steps"></a>Passos Seguintes

Antes de iniciar estes passos, tem de cumprir os [requisitos](./cpp-prerequisites.md) para publicar uma aplicação do Power BI no AppSource.
