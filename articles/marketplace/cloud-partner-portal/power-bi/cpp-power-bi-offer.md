# <a name="power-bi-app-offer"></a>Oferta da aplicação do Power BI

|              |                                |
|--------------|--------------------------------|
| Este artigo explica como publicar uma aplicação do Power BI da Microsoft [marketplace do AppSource](https://appsource.microsoft.com/).  Uma aplicação do Power BI pacotes de conteúdo personalizável do Power BI, incluindo conjuntos de dados, relatórios e dashboards. Pode, em seguida, implemente a aplicação para outros inquilinos do Power BI através do AppSource, efetuar a ajustes e personalizações permitidas pelo desenvolvedor e ligá-la aos seus próprios dados. | ![Ícone do Power BI](./media/powerbi-icon.png) |


Este artigo está dividido em três partes principais:

-   [Pré-requisitos](./cpp-prerequisites.md). Os requisitos técnicos e empresariais para criar e publicar uma oferta de aplicação do Power BI.
-   [Criar uma oferta de aplicação do Power BI](./cpp-create-offer.md). Como criar uma entrada de oferta de aplicação do Power BI com o [Cloud Partner Portal](https://cloudpartner.azure.com).
-   [Publicar uma oferta de aplicação do Power BI](./cpp-publish-offer.md). Como submeter no AppSource, uma nova oferta para publicação e como atualizar uma oferta existente.


## <a name="publishing-steps"></a>Passos de publicação

Aqui estão as etapas de alto nível para publicar uma oferta de aplicação do Power BI:

![Passos de publicação da oferta de aplicação Power BI](media/publishing-steps.png)

Esta é a oferta de aplicação do Power BI, processo de publicação:

1. Crie uma aplicação de modelo no Power BI. Esta ação gera um URL de instalação do pacote, que representa o elemento principal técnico para a oferta. Também neste momento, promova o pacote de teste para pré-produção. Para obter mais informações, consulte [o que são aplicações de modelo do Power BI?](https://docs.microsoft.com/power-bi/service-template-apps-overview). 
2. Recolher ou criar materiais de marketing da oferta, incluindo: nome oficial, descrição, logótipos, etc. 
3. Recolher ou criar legais da oferta e documentos de suporte: *termos de utilização*, *política de privacidade*, *política de suporte*, a ajuda do utilizador, etc.
4. Criar a oferta: utilizar o Portal de parceiro de Cloud para configurar os detalhes da oferta, incluindo a descrição da oferta, materiais, informações legais, informações de suporte e as especificações dos ativos de marketing.  Depois da oferta é totalmente especificada, submetê-lo para publicação.
5. Monitorize o processo de publicação no Portal de parceiros de nuvem.  Durante este passo, a equipa de integração do AppSource testa, valida e certifica que seu aplicativo. 
6. Depois da aplicação está certificada com êxito, examine-o em seu ambiente de teste e liberá-lo. 
7. A aplicação Power BI está listada no AppSource ("entram em direto"-lo.).
8. No Power BI, promova o pacote de pré-produção para produção. Para obter mais informações, consulte [gerenciar a versão do modelo de aplicação](https://docs.microsoft.com/power-bi/service-template-apps-create#manage-the-template-app-release).
