# <a name="team-data-science-process-lifecycle"></a>Ciclo de vida do processo de ciência de dados de equipa

O processo de ciência de dados de equipa (TDSP) fornece um ciclo de vida recomendado que pode utilizar para estruturar seus projetos de ciência de dados. O ciclo de vida descreve os passos, do início ao fim, que projetos siga, normalmente, quando estes são executados. Se estiver a utilizar o ciclo de vida das ciências de dados outro, tal como [CRISP-DM](https://wikipedia.org/wiki/Cross_Industry_Standard_Process_for_Data_Mining), [KDD](https://wikipedia.org/wiki/Data_mining#Process) ou o processo personalizado da organização, pode continuar a utilizar o TDSP baseado em tarefas com estes ciclos de vida de desenvolvimento. 

Este ciclo de vida tem foi concebido para projetos de ciência de dados que foram concebidos para são enviados como parte das aplicações inteligentes. Estas aplicações implementar machine learning ou artificial intelligence modelos para Análise Preditiva. Projetos de ciência de dados exploratórias e projetos de análise ad hoc também podem beneficiar da utilização deste processo. Mas nestes casos poderão não ser necessários alguns passos descritos.    

O ciclo de vida TDSP é constituído por cinco fases principais que são executados iteratively. Estas incluem:

* [1. a compreensão de negócio de](lifecycle-business-understanding.md)
* [2 aquisição de dados e a compreensão](lifecycle-data.md)
* [3. modelação de](lifecycle-modeling.md)
* [4. implementação de](lifecycle-deployment.md)
* [5. aceitação de cliente de](lifecycle-acceptance.md)

Eis uma representação visual do **ciclo de vida do processo de ciência de dados de equipa**. 

![Ciclo de vida TDSP](./media/lifecycle/tdsp-lifecycle2.png) 


O ciclo de vida do processo de ciência de dados de equipa é modelado como uma sequência de passos iterated que fornecem orientações sobre as tarefas necessárias para utilizar modelos preditivos. Estes modelos podem ser implementados num ambiente de produção para ser utilizado para criar aplicações inteligentes. O objetivo deste ciclo de vida do processo é continuar a mover um projeto de ciência reencaminhar do dados, para um ponto final do engagement encriptado. Apesar de ser VERDADEIRO se ciência de dados é um exercício na investigação e de deteção, ser capaz de comunicar claramente estas tarefas para a sua equipa e os seus clientes utilizando um conjunto bem definido de artefactos modelos padronizado os funcionários podem ajudar a evitar misunderstanding e aumentar a probabilidade de uma conclusão com êxito de um projeto de ciência de dados complexas.

Para cada fase, podemos fornecer as seguintes informações:

* **Objetivos**: os objetivos específicos.
* **Como fazê-lo**: a tarefas específicas, descritas e a orientação fornecida conclui-los.
* **Artefactos**: a concentrarem e o suporte para produzi-los.
