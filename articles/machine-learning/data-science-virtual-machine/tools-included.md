# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Que ferramentas estão incluídas na Máquina Virtual de Ciência de Dados Do Azure?

A Máquina Virtual da Ciência dos Dados é a forma mais fácil de explorar dados e fazer machine learning na nuvem. As Máquinas Virtuais da Ciência dos Dados são pré-configuradas com o sistema operativo completo, patches de segurança, controladores e software popular de ciência e desenvolvimento de dados. Você pode escolher o ambiente de hardware, que vai desde máquinas centradas em CPU de baixo custo a máquinas muito poderosas com múltiplas GPUs, armazenamento NVMe e grandes quantidades de memória. Para máquinas com GPUs, todos os controladores estão instalados, todos os quadros de machine learning são compatíveis com a versão para a compatibilidade com a GPU, e a aceleração está ativada em todo o software de aplicação que suporta GPUs.

A Máquina Virtual da Ciência dos Dados vem com as ferramentas de ciência de dados mais úteis pré-instaladas:

## <a name="how-to-choosecondaenvironment"></a>Como escolher o ambiente da conda  

> [!NOTE]
> Há mudanças de rutura ao mudar para a nova versão da Máquina Virtual da Ciência dos Dados a partir da versão antiga. 

**Mudar do Windows Server 2016 para windows Server 2019** </br> 

* O ambiente do condomínio está previsto para python37_default. Não apoiamos python2. Se estiver a usar tensorflow ou maçarico no ambiente de conda python, utilize py37_tensorflow ou py37_pytorch respectivamente. Se estiver a utilizar apenas o uso do AzureML azureml_py36_automl </br>

* Se estiver a utilizar tensorflow ou maçarico no ambiente de conda AzureML, utilize azureml_py36_tensorflow ou azureml_py36_pytorch respectivamente. </br>

**Passando de Ubuntu 16.04 para Ubuntu 18.04** </br>

* Se estiver a usar tensorflow ou maçarico no seu ambiente de conda azureml_py36, utilize azureml_py36_tensorflow ou azureml_py36_pytorch respectivamente. </br>

* Se estiver a usar tensorflow ou maçarico no seu ambiente de conda py37, use py37_tensorflow ou py37_pytorch respectivamente. </br>

* Caso contrário, utilize python37_default ou se estiver a utilizar apenas azureml_py36 utilizar azureml_py36_automl </br>
