# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Que ferramentas estão incluídas na Máquina Virtual de Ciência de Dados do Azure?

A Máquina Virtual da Ciência de Dados é uma forma fácil de explorar dados e fazer machine learning na nuvem. As Máquinas Virtuais da Ciência de Dados estão pré-configuradas com o sistema operativo completo, patches de segurança, controladores e software popular de ciência e desenvolvimento de dados. Você pode escolher o ambiente de hardware, que vai de máquinas centradas em CPU de baixo custo a máquinas muito poderosas com várias GPUs, armazenamento NVMe e grandes quantidades de memória. Para máquinas com GPUs, todos os controladores são instalados, todos os quadros de machine learning são compatíveis com a versão para compatibilidade gpu, e a aceleração é ativada em todos os softwares de aplicação que suportam GPUs.

A Máquina Virtual da Ciência de Dados vem com as ferramentas de ciência de dados mais úteis pré-instaladas.

## <a name="how-to-choosecondaenvironment"></a>Como escolher o ambiente conda  

> [!NOTE]
> Há mudanças de rutura na mudança para a nova versão da Data Science Virtual Machine a partir da versão antiga. 

**Mudar de Windows Server 2016 para Windows Server 2019** </br> 

* O ambiente conda está definido para python37_default. Não apoiamos a pitão2. Se estiver a utilizar o TensorFlow ou o pytorch no seu ambiente python conda, utilize py37_tensorflow ou py37_pytorch respectivamente. Se estivesse a utilizar apenas a AzureML, azureml_py36_automl </br>

* Se estiver a utilizar o TensorFlow ou o pytorch no seu ambiente AzureML conda use azureml_py36_tensorflow ou azureml_py36_pytorch respectivamente. </br>

**Mudar de Ubuntu 16.04 para Ubuntu 18.04** </br>

* Se estiver a utilizar o TensorFlow ou o pytorch no seu ambiente conda azureml_py36, utilize azureml_py36_tensorflow ou azureml_py36_pytorch respectivamente. </br>

* Se estiver a utilizar o TensorFlow ou o pytorch no seu ambiente py37 conda use py37_tensorflow ou py37_pytorch respectivamente. </br>

* Caso contrário, utilize python37_default ou se utilizasse apenas azureml_py36 utilizar azureml_py36_automl </br>
