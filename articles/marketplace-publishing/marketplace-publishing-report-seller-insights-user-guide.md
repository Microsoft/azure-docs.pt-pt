# <a name="how-to-use-the-azure-marketplace-seller-insights-portal"></a>Como utilizar o portal do Azure Marketplace vendedor Insights

Este artigo descreve como os administradores de aceder ao portal vendedor Insights, adicionar ou eliminar utilizadores e definir as permissões para utilizadores.

## <a name="onboard-users"></a>Utilizadores carregar

Esta secção descreve como os administradores de aceder ao portal de vendedor Insights pela primeira vez. As tarefas nesta secção estão reservadas para administradores. 


### <a name="overview-of-roles"></a>Descrição geral das funções

O administrador do portal vendedor Insights adiciona utilizadores, edita os utilizadores e elimina utilizadores.

Um utilizador de portal vendedor Insights pode ver todos os dados que está disponíveis para a sua organização.

Os administradores e utilizadores são necessárias para autenticar usando um [conta Microsoft](http://signup.live.com), ou a organizações conta escolar ou profissional. Certifique-se de que o novo ID de conta está associado uma caixa de correio monitorizada.

![Tipo de conta][5]



### <a name="invitation-to-activate-your-account"></a>Convite para ativar a sua conta

Antes de iniciar sessão no portal do vendedor Insights, o administrador deve identificar as contas de indivíduos que pretendem configurar como os utilizadores. Configure utilizadores com uma ou conta profissional ou uma conta Microsoft.

Para ativar o seu serviço, o administrador deve passar para o [endereço de URL](https://reports.azure.com) que aparece no e-mail de convite.

>[!NOTE]
>Se o e-mail de administrador está indicado como uma conta Microsoft e não tiver criado uma conta Microsoft que está associado com o endereço de e-mail a partir da qual foi recebido o convite, terá de [criar a conta](https://signup.live.com) antes de iniciar sessão.


### <a name="administrator-sign-in-to-the-seller-insights-portal"></a>Administrador inicie sessão no portal do vendedor Insights

1. Inicie sessão no portal do vendedor Insights clicando no URL do e-mail que é enviado.
2. Introduza o endereço de correio eletrónico do administrador e a palavra-passe.
3. Clique em de **sessão** botão.
4. Begin para administrar a sua conta. Adicionar utilizadores, definir as permissões e editar utilizadores.
5. Certifique-se de que os utilizadores têm o URL para o portal de vendedor Insights.

    ![Página de início de sessão][6]

## <a name="user-sign-in-to-the-portal"></a>Utilizador inicie sessão no portal

O administrador deve notificá-lo através de e-mail que a sua conta está criada e a partilha a [endereço de URL](https://reports.azure.com) para o portal de vendedor Insights consigo.

>[!NOTE]
>O administrador é o proprietário da conta que publica soluções para o Azure Marketplace. O proprietário desta conta pode fornecer acesso a outras pessoas.  Se tiver acesso ao portal de publicação como coadministrador, pode encontrar o endereço de administrador, seguindo estes passos:
>1. Inicie sessão no portal de publicação.
>2. Clique em de **publicadores** separador.
>3. Abra o publicador em questão (normalmente, não há apenas um).
>4. Clique em **administradores** na parte inferior esquerda.
>5. O endereço de e-mail do administrador que procura assinalada como *proprietário*.


Pode iniciar sessão no portal do vendedor Insights utilizando uma conta Microsoft pessoal, ou utilizando uma conta escolar ou profissional. O tipo de conta depende do que o administrador configura a no seu perfil. O administrador deve aconselhe o processo sobre a conta que utiliza.

![Tipo de conta][1]



**Bem-vindo ao portal do vendedor Insights!**

1. Depois de ter sessão iniciada, é apresentado no **resumo** página.
2. Demore alguns minutos para ler a política de privacidade.
3. Pode **contacte-na** em qualquer altura, através de e-mail, com perguntas poderá ter que estão relacionados com o portal do vendedor Insights.
4. Pode mover-se no portal do vendedor Insights utilizando os separadores na parte superior da página.  Procure os separadores na parte superior das páginas neste guia para ajudar a incluída, no portal do vendedor Insights.
5. Clique no ícone de smile em qualquer altura para enviar comentários relacionados com o portal do vendedor Insights.
6. Para terminar sessão no portal do vendedor Insights, clique em seu nome no canto superior direito.

    ![Página de destino predefinido][2]


## <a name="summary-view"></a>Vista de resumo


1. Selecione o **resumo** separador.
2. Introduza o **data de início** do período de tempo que interessa aqui e, em seguida, clique em **aplicar**. São mostrados durante três meses de dados.
3. Novos dados de tendências de ordem para o período de tempo de três meses que selecionou são apresentados aqui.
4. As ordens de novo para o período de tempo selecionado são mostradas aqui por canal e a oferecem de tipo.
5. Selecione uma vista este painel. **Parte superior 5 mercado**, **principais 5 ofertas efetuar**, ou **principais 5 clientes**.
6. Tendência de dividendos selecionado por período de tempo é mostrada aqui para ofertas de pagas. Se tiver apenas BYOL ou a partir do dia 1 terceiros ofertas, não é apresentada neste painel.

    ![Vista de resumo][7]

No *painel A*, pode selecionar o **encomendas pendentes e utilização** separador.  Aqui pode selecionar **ordens** ou **utilização** de **tipo de relatório**e, em seguida, utilize os controlos de deslize para selecionar os dados de interesse.

No *painel C*, pode selecionar os dados de tendências de três meses. Selecione **ordens** ou **utilização** dependendo da seleção no **tipo de relatório** no painel da.  Os dados apresentados refletem as datas que introduzir e os filtros aplicados ao utilizar os controlos de deslize no painel da.

No *painel a*, os dados de subscrição para cada indivíduo estão listados.  Dados estão disponível para transferência a partir deste painel no formato de Excel ou CSV.

## <a name="user-management"></a>Gestão de utilizadores

As tarefas nesta secção estão reservadas para administradores. Utilize os procedimentos seguintes para adicionar utilizadores, alterar as permissões ou eliminar os utilizadores a partir do portal do vendedor Insights.


### <a name="add-user"></a>Adicionar utilizador

Esta secção mostra como adicionar novos utilizadores ou alterar as respetivas permissões no portal do vendedor Insights.


1. Selecione o **gestão de utilizadores** separador.
2. Para adicionar um utilizador, selecione **adicionar utilizador**.
3. Introduza o endereço de e-mail que está associado a respetiva conta Microsoft ou a respetiva conta escolar ou profissional. Em seguida, clique em **continuar**.
4. Como administrador, pode atribuir acesso de relatório só de leitura aos utilizadores.
5. Introduza os detalhes de utilizador, **nome próprio**, **Apelido**, e **E-Mail**e, em seguida, clique em **continuar**.
6. Seleccionar as permissões que pretende alocar para o utilizador e, em seguida, clique em **continuar**.
7. Certifique-se de que as informações que introduziu estão corretas e, em seguida, clique em **guardar** para continuar.  Adicionou com êxito um utilizador.

  ![Adicionar um painel de utilizador][29]


### <a name="edit-user"></a>Editar utilizador

1. Selecione o **gestão de utilizadores** separador e, em seguida, selecione **editar utilizadores**.
2. Localizar o utilizador cujas permissões que pretende modificar.  Também pode adicionar novos utilizadores. Clique em **adicionar utilizadores**.
3. Clique nas permissões que pretende remover o utilizador.  Também pode eliminar um utilizador. Clique em **eliminar utilizador**.
4. Clique em **guardar** para continuar.  Tiver editada com êxito as permissões do utilizador.

  ![Editar um painel de utilizador][30]

## <a name="help-support-requests"></a>Pedidos de suporte de ajuda


### <a name="access-the-support-request-format"></a>O formato do pedido de suporte de acesso

1. Selecione o **ajudar** separador.
2. Selecione **suporte & documentação**e, em seguida, selecione **iniciar pedido**.
3. Também pode submeter um pedido de suporte do **contacte-na** ligação. Esta ligação é apresentada na parte inferior de cada página na aplicação.
4. Um novo separador abre-se onde pode submeter um pedido de suporte.

  ![Acesso a forma de suporte][31]

### <a name="review-helpful-information"></a>Rever informações úteis

1. Para submeter um pedido de suporte que está relacionado com os seus relatórios, selecione **relatórios e Insights** de **tipo de problema**.
2. Seleccione uma categoria.  As opções são **perguntas relacionadas com relatórios** e **outros**.
3. São apresentadas algumas hiperligações úteis, que pode fornecer uma resposta à sua consulta.  Verifique estas ligações antes de submeter o pedido de suporte.
4. Se ainda precisar de submeter um pedido de suporte, selecione **iniciar pedido**.

  ![Editar um painel de utilizador][32]


### <a name="enter-information-and-submit-the-request"></a>Introduza as informações e submeter o pedido

1. Introduza as informações de contacto, fuso horário, região e preferências de idioma e, em seguida, selecione **continuar**.
2. Atribuir o incidente um título e fornecer maior quantidade informações quanto possível.  Pode também carregar ficheiros de suporte, como uma captura de ecrã e, em seguida, selecione **submeter**.
3. Receberá uma confirmação de que o seu pedido de suporte é efetuada com êxito.
4. Pode verificar o estado do seu pedido de suporte em qualquer altura.

  ![Acesso a forma de suporte][33]


## <a name="reset-a-password"></a>Repor uma palavra-passe

**Validação de conta e palavra-passe**

1. *Confirme que a conta que está a utilizar está registada e a palavra-passe não funciona.* Se introduzir uma conta que está registada no Azure Active Directory, é automaticamente pede-lhe para escolher como iniciar sessão com a conta. Utilize um ou conta profissional ou uma conta Microsoft que serve como a sua conta pessoal.
2. Aceda a [https://reports.azure.com](https://reports.azure.com) e início de sessão fora de qualquer conta que poderá ser com que iniciou sessão. Também pode abrir numa janela privada para garantir que não existem outras contas com sessão iniciadas.
3. Introduza o endereço de correio eletrónico do publicador, mas não introduzir uma palavra-passe. Se está registado no Azure Active Directory, irá redirecionar para o passo 4.

    >[!NOTE]
    >As contas de administrador são registadas. As contas escolares ou profissionais que são definidas pela conta de administrador também estão registadas.

4. Escolha **trabalho conta escolar ou profissional**. Contas escolares ou profissionais são configuradas por empresas que estão registadas com o Azure. 

    >[!NOTE]
    >É possível que um utilizador configurar uma conta Microsoft utilizando o endereço de correio eletrónico da sua organização. Neste cenário, a organização onboards posterior para o Azure e o mesmo endereço de correio eletrónico está registado como uma conta escolar ou profissional. Este utilizador pode ter uma conta Microsoft com a mesma palavra-passe que podem ser utilizados para criar a conta escolar ou profissional. Este utilizador também pode ter uma conta escolar ou profissional com uma palavra-passe separada que é utilizada pela sua organização.

5. Para o portal de vendedor Insights, quando seleciona **trabalhar conta escolar ou profissional**, certifique-se de que utiliza o seu trabalho ou palavra-passe de conta profissional e não uma conta palavra-passe Microsoft.
6. Uma conta escolar ou profissional redireciona-o para a página do publicador para iniciar sessão.
7. Aqui, confirme que a palavra-passe para a conta escolar ou profissional funciona. Se não funcionar, continue para a secção "Repor palavra-passe" deste artigo.



**Repor palavra-passe**

Utilize os seguinte da Microsoft e contas profissionais ou escolares para repor a palavra-passe.

>[!NOTE]
>Para as contas escolares ou profissionais que são definidas pela conta de administrador, se o administrador ativada a capacidade para permitem-lhe repor a sua própria palavra-passe, pode repor a sua própria palavra-passe. Caso contrário, o formulário de reposição de palavra-passe dá-lhe a opção para contactar o administrador.

* [Conta Microsoft](https://account.live.com/ResetPassword.aspx)
![repor a palavra-passe de conta Microsoft][3]

* [Conta escolar ou profissional](https://passwordreset.microsoftonline.com/)
![conta escolar ou profissional de reposição de palavra-passe][4]
