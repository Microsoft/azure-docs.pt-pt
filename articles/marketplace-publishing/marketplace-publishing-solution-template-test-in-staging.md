# <a name="test-your-solution-template-offer-in-staging"></a>Testar a sua oferta de modelo de solução para efeitos de teste
Fase de testes significa implementar a sua oferta no "sandbox privada" onde pode testar e verificar a funcionalidade dele antes de enviá-la para produção. A oferta é apresentada no teste tal como faria para um cliente que tenha implementado. A oferta deve ser certificada para ser emitida para teste.

Depois da oferta é testada, pode ver e testar a oferta no [Portal do Azure](https://portal.azure.com/).

Siga os passos abaixo para emitir a oferta para teste e testá-lo no [Portal do Azure](https://portal.azure.com/):

1. Vá para o [Portal de publicação](https://publish.windowsazure.com) > **modelos de soluções** separador > sua oferta > **publicar** > **emitir para teste**.
2. Fornece a lista de subscrições do Azure que irá utilizar para pré-visualizar e testar a sua oferta.
3. Inicie sessão no portal de pré-visualização do Azure com o ID de subscrição que utilizou no passo anterior.
4. Execute, pelo menos, uma etapa do teste no portal de pré-visualização do Azure nos pontos de mencionadas a seguir:
   * Certifique-se de que conteúdo de marketing aparece corretamente no Azure Marketplace.
   * Implementação de ponto a ponto da topologia.
   * Execute testes de desempenho e testes de estresse.
   * Certifique-se de que a topologia respeita as melhores práticas.

## <a name="next-steps"></a>Passos Seguintes
Se estiver satisfeito com os resultados, em seguida, pode avançar para a fase de publicação de oferta final, **passo 4**: [implementar a sua oferta no Marketplace](marketplace-publishing-push-to-production.md). Caso contrário, fazer alterações na sua oferta e solicitar a certificação novamente.

> [!NOTE]
> Para alterações de conteúdo de marketing, a certificação não é necessária.
