[Como definir a segurança no DevTest Labs](https://channel9.msdn.com/Blogs/Azure/How-to-set-security-in-your-DevTest-Lab)

Neste vídeo, vamos mostrar como adicionar uma pessoa em qualquer uma destas funções a um laboratório.

[Gerir segredos para aplicações do Azure](https://channel9.msdn.com/events/Build/2016/P456)

Todas as aplicações grave, que implementa no Azure têm críticos segredos – cadeias de ligação, certificados, chaves. Erros tolice gerir estes segredos leva a fatais conseqüências – vazamentos, falhas, violações de conformidade. Como vários pesquisas recentes menciona o tempo limite, erros de tolice causam quatro vezes mais violações de dados que os adversários. Nesta sessão, vamos abordar algumas das melhores práticas para gerir os segredos de aplicações importantes. Essas práticas recomendadas podem parecer senso comum, no entanto, muitos desenvolvedores negligenciá-los. Vamos também sobre como utilizar o Azure Key Vault para implementar essas práticas recomendadas. Como vantagem adicional, ao seguir estas práticas recomendadas de ajuda demonstrar a conformidade com padrões, como SOC. Os primeiros 10 minutos da sessão são nível 100 e estas são aplicadas a qualquer aplicação na cloud que desenvolver em qualquer plataforma. O restante é o nível 200 e 300 e se concentra em aplicações que cria na plataforma do Azure.

Gerente de programas Sênior Narayan Annamalai ensina Scott a utilizar grupos de segurança de rede dentro de uma rede Virtual do Azure. É possível criar a controlar o acesso a objetos no Azure por sub-rede e rede! Saiba como controlar o acesso e criar grupos no Azure com o PowerShell.

* Monitorização e corrigir as descobertas de segurança
* Rever o que precisa de permanecer nas funções privilegiadas para fluxos de trabalho recertification periódica
