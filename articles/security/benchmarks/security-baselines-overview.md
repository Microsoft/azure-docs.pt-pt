# <a name="security-baselines-for-azure"></a>Linhas de base de segurança do Azure

As linhas de segurança para o Azure ajudam-no a fortalecer a segurança através de funcionalidades de ferramentas, rastreios e segurança melhorados. Eles também lhe proporcionam uma experiência consistente ao garantir o seu ambiente.

As linhas de segurança para o Azure concentram-se em áreas de controlo centradas na nuvem. Estes controlos são consistentes com referências de segurança bem conhecidas, como as descritas pelo Center for Internet Security (CIS). As nossas linhas de base fornecem orientação para as áreas de controlo listadas no [Azure Security Benchmark](overview.md).

Cada recomendação inclui as seguintes informações:

- **Azure ID**: O ID de benchmark de segurança Azure que corresponde à recomendação.
- **Recomendação**: Seguindo imediatamente após o Azure ID, a recomendação fornece uma descrição de alto nível do controlo.
- **Orientação**: A lógica da recomendação e as ligações à orientação sobre como implementá-la. Se a recomendação for apoiada pelo Azure Security Center, essa informação também será listada.
- **Responsabilidade**: Quem é responsável pela implementação do controlo. Os cenários possíveis são a responsabilidade do cliente, a responsabilidade da Microsoft ou a responsabilidade partilhada.
- **Monitorização do Centro de Segurança Azure**: Se o controlo é monitorizado pelo Azure Security Center, com ligação à referência.

Todas as recomendações, incluindo recomendações que não são aplicáveis a este serviço específico, estão incluídas na linha de base para fornecer-lhe uma imagem completa de como o Azure Security Benchmark se relaciona com cada serviço. Pode ocasionalmente haver controlos que não são aplicáveis por várias razões — por exemplo, os controlos iaaS/centrados no cálculo (tais como controlos específicos da gestão da configuração do SO) podem não ser aplicáveis aos serviços paaS.
