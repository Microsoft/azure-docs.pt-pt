# <a name="a-practical-guide-to-designing-secure-health-care-solutions-in-azure"></a>Um guia prático para criar soluções de assistência médica seguras no Azure
As inicializações do setor de saúde, SIs (integradores de sistema), ISVs (fornecedores independentes de software) e organizações de saúde que consideram uma mudança para o Azure estão procurando diretrizes que ajudam a incorporar controles de segurança para atender às suas obrigações de conformidade.

[Um guia prático para criar soluções de assistência médica seguras no Microsoft Azure](https://aka.ms/azureindustrysecurity) ajuda a entender como você pode melhorar a segurança para suas soluções usando os serviços e recursos do Azure que você pode configurar com base em seus requisitos.
O conteúdo é dividido em três seções principais:

1. Diretrizes de considerações para uso da tecnologia de nuvem, incluindo gerenciamento de riscos, responsabilidade compartilhada, estabelecimento de um sistema de gerenciamento de segurança de informações, compreensão de regulamentos locais e do setor e estabelecimento de procedimentos operacionais padrão.
2. Os principais princípios de segurança que estão alinhados a um padrão de gerenciamento de segurança de informações padrão, como ISO 27001, e processos de desenvolvimento padrão, como o SDL (ciclo de vida de desenvolvimento de segurança) da Microsoft.
3. Aplicar os principais princípios aos casos de uso demonstrando o alinhamento de uma perspectiva do arquiteto de soluções, em que os requisitos das soluções estão alinhados ao padrão de gerenciamento de segurança de informações.

Esperamos que você encontre [um guia prático para criar soluções de assistência médica seguras](https://aka.ms/azureindustrysecurity) úteis e, se tiver dúvidas ou sugestões, fale conosco deixando um comentário abaixo.
