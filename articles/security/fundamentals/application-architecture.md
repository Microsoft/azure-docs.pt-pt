# <a name="application-architecture-on-azure"></a>Arquitetura de aplicações no Azure
Para ajudar a proteger suas soluções baseadas em nuvem no Microsoft Azure, uma base de arquitetura forte é fundamental. Arquitetos, designers e implementadores todos se beneficiam de um forte conhecimento de arquitetura de aplicativos e serviços. Esse conhecimento básico ajuda você a compreender todos os componentes de suas soluções baseadas em nuvem e a facilitar a integração da segurança em todos os aspectos do design e da implementação.

Temos o seguinte para ajudá-lo com suas investigações e designs de arquitetura:

* Plantas arquitetônicas
* Conjunto de símbolos de nuvem e Enterprise
* modelo do Visio para o plano 3D

## <a name="architectural-blueprints"></a>Plantas arquitetônicas
A Microsoft publica um conjunto de [plantas de arquitetura](https://aka.ms/azblueprints) de alto nível, mostrando como criar tipos específicos de sistemas usando produtos da Microsoft.
Cada projeto inclui um:

* Arquivo com base em 2D do Visio 2003 que você pode baixar e modificar
* Arquivo PDF de perspectiva 3D colorida para apresentar o plano gráfico a menos para os públicos técnicos
* Vídeo que percorre a versão 3D

## <a name="cloud-and-enterprise-symbol-set"></a>Conjunto de símbolos de nuvem e Enterprise
[Exiba o vídeo treinamento de símbolos e Visio](https://aka.ms/CnESymbolsVideo) e, em seguida, [Baixe o conjunto de símbolos de nuvem e Enterprise](https://aka.ms/CnESymbols) para ajudar a criar materiais técnicos que descrevem o Azure, o Windows Server, SQL Server e muito mais. Você pode usar os símbolos em diagramas de arquitetura, materiais de treinamento, apresentações, folhas de documentos, infográficos, White papers e até mesmo livros de terceiros se o livro treinar as pessoas a usar os produtos da Microsoft. No entanto, elas não são destinadas para uso em interfaces do usuário.

## <a name="3d-blueprint-visio-template"></a>modelo do Visio para o plano 3D
As versões 3D dos [planos gráficos de arquitetura da Microsoft](https://aka.ms/azblueprints) foram inicialmente criadas em uma ferramenta que não é da Microsoft. Um novo modelo do Visio 2013 (e posterior) fornecido em 5 de agosto de 2015 como parte de um [curso de certificação de arquitetura da Microsoft distribuído em EDX.org](https://docs.microsoft.com/azure/architecture/).

O modelo também está disponível fora do curso.

* [Veja primeiro o vídeo de treinamento](https://aka.ms/3dBlueprintTemplateVideo) para que você saiba o que ele pode fazer
* Baixe o [modelo Microsoft 3D Blueprint Visio](https://aka.ms/3DBlueprintTemplate)
* Baixar os [símbolos de nuvem e Enterprise](https://docs.microsoft.com/azure/architecture/) para usar com o modelo 3D
