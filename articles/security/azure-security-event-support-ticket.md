# <a name="how-to-log-a-security-event-support-ticket"></a>Como iniciar um pedido de suporte de evento de segurança

1. Navegue para [suporte do Editor](https://support.microsoft.com/en-us/getsupport?wf=0&tenant=ClassicCommercial&oaspworkflow=start_1.0.0.0&locale=en-us&supportregion=en-us&pesid=16230&ccsid=636450758943226673) e inicie sessão com as credenciais da Microsoft.
2. Selecione "Evento de segurança" como o tipo de problema e escolha entre as categorias de "Incidente de segurança" e "Vulnerabilidade".

    ![Tipo de evento e a definição][1]

3. Depois de selecionar o tipo de problema e a categoria, clique em do '**pedido de início de**' botão. Forneça as seguintes informações para ajudar-na compreender melhor o problema.

    i. O que é o problema e/ou a vulnerabilidade?

    ii. Vulnerabilidades, forneça a CVE (mitre.org) ou o preenchido horizontalmente CVSS3 v3 Calculadora (https://www.first.org/cvss/calculator/3.0).

    III. Existe uma resolução ou atenuação? Se Sim, em seguida, forneça os passos de remediação.

    IV. Tem uma mensagem que pretende enviar para os clientes? Iremos trabalhar consigo para criar uma mensagem apropriada se aplicável.

4. Confirmação de submissão - depois de ter submetido o seu problema, iremos confirmar o recebimento num dia útil e seu problema de atribuir uma prioridade e gravidade.

    - Se precisar de comunicar com o-nos sobre o seu problema, utilize o número de confirmação em todas as correspondências.
    - Pode ver o progresso no seu problema em qualquer altura.

5. O que acontece depois? Dependendo do problema e a gravidade, podem ser demorados os seguintes passos:

    - Iremos comunicar o resultado de nossa avaliação para. Consoante o resultado, poderemos remover ou que modificar a sua oferta de pedido. Neste evento, iremos trabalhar consigo para se certificar de que a interrupção para os clientes afetados é minimizada.
    - Iremos trabalhar consigo para ajudar a atenuar o impacto do incidente/vulnerabilidade para os nossos clientes mútuos.
