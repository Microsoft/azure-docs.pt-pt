# <a name="dedicated-sql-pool-formerly-sql-dw-in-azure-synapse-analytics-release-notes"></a>Piscina SQL dedicada (anteriormente SQL DW) em notas de lançamento do Azure Synapse Analytics

Este artigo resume as novas funcionalidades e melhorias nos recentes lançamentos de [pool SQL (anteriormente SQL DW)](sql-data-warehouse-overview-what-is.md) em Azure Synapse Analytics. O artigo também lista notáveis atualizações de conteúdo que não estão diretamente relacionadas com o lançamento, mas publicadas no mesmo período de tempo. Para melhorar outros serviços Azure, consulte [as atualizações do Serviço](https://azure.microsoft.com/updates).

## <a name="check-your-dedicated-sql-pool-formerly-sql-dw-version"></a>Verifique a sua versão dedicada da piscina SQL (anteriormente SQL DW)

À medida que as novas funcionalidades são lançadas para todas as regiões, verifique a versão implementada no seu caso e as últimas notas de lançamento para disponibilidade de funcionalidades. Para verificar a versão, ligue-se à sua piscina DE SQL (anteriormente SQL DW) via SQL Server Management Studio (SSMS) e corra `SELECT @@VERSION;` para devolver a versão atual. Utilize esta versão para confirmar que lançamento foi aplicado na sua piscina SQL dedicada (anteriormente SQL DW). A data na saída identifica o mês para o lançamento aplicado à sua piscina SQL dedicada (anteriormente SQL DW). Isto aplica-se apenas às melhorias ao nível do serviço. 

Para melhorias na ferramenta, certifique-se de que tem a versão correta instalada na nota de lançamento. 

> [!NOTE]
> O nome do produto devolvido por SELECT @ @VERSION mudará de Microsoft Azure SQL Data Warehouse para Microsoft Azure Synapse Analytics. Enviaremos um aviso prévio antes da mudança ser feita. Esta alteração é relevante para os clientes que analisam o nome do produto a partir do resultado do SELECT @ @VERSION no seu código de aplicação. Para evitar alterações no código de aplicação devido ao rebranding do produto, utilize estes comandos para consultar o SERVERPROPERTY para o nome e versão do produto da base de dados: Para devolver o número da versão XX.X.XXXXX.X (sem nome do produto) utilize este comando:
>
> ```sql
> SELECT SERVERPROPERTY('ProductVersion')
>
> --To return engine edition, use this command that returns 6 for Azure Synapse Analytics:
>
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Da mesma forma, a exportação de dados de Azure Synapse para Azure Storage garantidos para VNet também é suportado através da Polybase.<br/><br/>Para obter mais informações sobre os pontos finais do serviço VNet em Azure Synapse, consulte o post do [blog](https://azure.microsoft.com/blog/general-availability-of-vnet-service-endpoints-for-azure-sql-data-warehouse/) ou a [documentação](../../azure-sql/database/vnet-service-endpoint-rule-overview.md?toc=/azure/synapse-analytics/sql-data-warehouse/toc.json&bc=/azure/synapse-analytics/sql-data-warehouse/breadcrumb/toc.json).|
|**Monitorização automática do desempenho (pré-visualização)**|[A Loja de Consultas](/sql/relational-databases/performance/monitoring-performance-by-using-the-query-store?toc=/azure/synapse-analytics/sql-data-warehouse/toc.json&bc=/azure/synapse-analytics/sql-data-warehouse/breadcrumb/toc.json&view=azure-sqldw-latest&preserve-view=true) está agora disponível em Preview em SQL Analytics em Azure Synapse. A Consulta Store foi concebida para o ajudar a resolver problemas de desempenho de consulta, rastreando consultas, planos de consulta, estatísticas de tempo de execução e histórico de consultas para ajudá-lo a monitorizar a atividade e o desempenho do seu armazém de dados. A Query Store é um conjunto de lojas internas e Vistas dinâmicas de gestão (DMVs) que lhe permitem:<br/><br/>&bull;&nbsp;Identificar e afinar consultas de consumo de recursos de topo<br/>&bull;&nbsp;Identificar e melhorar cargas de trabalho não planeadas<br/>&bull;&nbsp;Avaliar o desempenho da consulta e o impacto no plano através de alterações nas estatísticas, índices ou tamanho do sistema (definição de DWU)<br/>&bull;&nbsp;Consulte o texto de consulta completa para todas as consultas executadas<br/><br/>A Loja de Consultas contém três lojas reais:<br/>&bull;&nbsp;Uma loja de planos para persistir a informação do plano de execução<br/>&bull;&nbsp;Uma loja de estatísticas de tempo de execução para persistir nas estatísticas de execução<br/>&bull;&nbsp;Uma loja de estatísticas de espera para informações de estatísticas de espera persistentes.<br/><br/>A SQL Analytics em Azure Synapse gere estas lojas automaticamente e fornece um número ilimitado de consultas contadas ao longo dos últimos sete dias sem custo adicional. Habilitar a Loja de Consultas é tão simples como executar uma declaração DE T-SQL ALTER DATABASE: <br/>sql ----ALTER DATABASE [DatabaseName] SET QUERY_STORE = ON;-------Para obter mais informações sobre a Loja de Consultas, consulte o artigo, [o desempenho da monitorização utilizando a Loja de Consultas,](/sql/relational-databases/performance/monitoring-performance-by-using-the-query-store)e os DMVs da Loja de Consultas, como [sys.query_store_query](/sql/relational-databases/system-catalog-views/sys-query-store-query-transact-sql?toc=/azure/synapse-analytics/sql-data-warehouse/toc.json&bc=/azure/synapse-analytics/sql-data-warehouse/breadcrumb/toc.json&view=azure-sqldw-latest&preserve-view=true). Aqui está o post do [blog](https://azure.microsoft.com/blog/automatic-performance-monitoring-in-azure-sql-data-warehouse-with-query-store/) anunciando o lançamento.|
|**Níveis de computação mais baixos para a SQL Analytics**|A SQL Analytics em Azure Synapse suporta agora níveis de computação mais baixos. Os clientes podem experimentar o desempenho, flexibilidade e funcionalidades de segurança principais da Azure Synapse, começando com 100 cDWU[(unidades de armazém de dados)](what-is-a-data-warehouse-unit-dwu-cdwu.md)e escala para 30.000 cDWU em minutos. A partir de meados de dezembro de 2018, os clientes podem beneficiar do desempenho e flexibilidade da Gen2 com níveis de computação mais baixos nas [regiões,](gen2-migration-schedule.md#automated-schedule-and-region-availability-table)com as restantes regiões disponíveis durante 2019.<br/><br/>Ao deixar cair o ponto de entrada para o armazenamento de dados da próxima geração, a Microsoft abre as portas a clientes orientados para o valor que querem avaliar todos os benefícios de um armazém de dados seguro e de alto desempenho sem adivinhar qual o ambiente de teste melhor para eles. Os clientes podem começar a partir de 100 cDWU, abaixo do atual ponto de entrada de 500 cDWU. A SQL Analytics continua a apoiar as operações de pausa e retomada e vai além da flexibilidade no cálculo. A Gen2 também suporta capacidade de armazenamento ilimitado de lojas de colunas, juntamente com 2,5 vezes mais memória por consulta, até 128 consultas simultâneas e funcionalidades [de cache adaptativas.](https://azure.microsoft.com/blog/adaptive-caching-powers-azure-sql-data-warehouse-performance-gains/) Estas funcionalidades, em média, trazem cinco vezes mais desempenho em comparação com a mesma Unidade de Armazém de Dados na Gen1 ao mesmo preço. As cópias de segurança geo-redundantes são padrão para a Gen2 com proteção de dados garantida incorporada. A SQL Analytics em Azure Synapse está pronta para escalar quando estiver.|
|**Fusão de fundo de colunas**|Por predefinição, o Azure SQL Data armazena dados em formato colunar, com micropartições chamadas [grupos de linha](sql-data-warehouse-memory-optimizations-for-columnstore-compression.md). Por vezes, devido a restrições de memória no índice de construção ou tempo de carga de dados, os grupos de linha podem ser comprimidos com menos do que o tamanho ideal de um milhão de linhas. Os grupos de linha também podem ficar fragmentados devido a eliminações. Grupos de linha pequenos ou fragmentados resultam num maior consumo de memória, bem como na execução de consultas ineficientes. Com esta libertação, a tarefa de manutenção de fundo de colunas combina pequenos grupos de linha comprimidos para criar grupos de linha maiores para melhor utilizar a memória e acelerar a execução de consultas.
