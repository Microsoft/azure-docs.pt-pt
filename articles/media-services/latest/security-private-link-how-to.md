# <a name="create-a-media-services-and-storage-account-with-a-private-link"></a>Criar uma conta de Media Services e Armazenamento com uma Ligação Privada

[!INCLUDE [media services api v3 logo](./includes/v3-hr.md)]

Criar uma conta de Serviços de Mídia e Conta de Armazenamento com Links Privados para um VNet. O modelo Azure Resource Manager (ARM) também configura o DNS para ambos os Links Privados. Finalmente, o modelo cria um VM para permitir ao utilizador experimentar os Links Privados.

## <a name="prerequisites"></a>Pré-requisitos

Leia [Quickstart: Crie e implemente os modelos ARM utilizando o portal Azure](https://docs.microsoft.com/azure/azure-resource-manager/templates/quickstart-create-templates-use-the-portal).

## <a name="limitations"></a>Limitações

- Para os Serviços de Mídia, o modelo apenas configura o Link Privado para a Entrega de Chaves.
- Um grupo de segurança de rede não é criado para o VM.
- O controlo de acesso à rede não está configurado para a Conta de Armazenamento ou Entrega de Chaves.

O modelo cria:

- Uma conta de Serviços de Mídia e uma Conta de Armazenamento (normalmente)
- Um VNet com uma sub-rede
- Tanto para a conta dos Serviços de Mídia como para a Conta de Armazenamento:
  - Pontos Finais Privados
  - Zonas privadas de DNS
  - Ligações entre ligações (para ligar as zonas privadas de DNS ao VNet)
  - Grupos privados de zona de DNS (para desencadear a criação automática de registos DNS nas zonas privadas de DNS)
- Um VM (com endereço IP público associado e interface de rede)
